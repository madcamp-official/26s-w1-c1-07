## §1. 디자인 시스템 — "Neo-Brutal Duel Arena"

컨셉 한 줄: **Gumroad풍 네오브루탈리즘 × 격투게임 HUD.** 크림 배경 위에 두꺼운 검정 보더,
blur 없는 하드 드롭섀도, 충돌하는 원색 블록, 과대한 대문자 타이포. '버튼 2개짜리 초간단 대결'을
스크린샷 한 장으로 각인시키는 밈 친화 룩.

### 1.3 형태 언어

- **radius**: 기본 `0px`. 예외 — 키캡·배지·pill 요소만 `8px`. 원형은 프로필 아바타·설정 버튼만.
- **border**: 모든 박스 `3px solid var(--ink)`. 히어로 요소(메인 버튼, 인게임 스테이지, 모달)는 `4px`.
  디바이더는 `2px`. 1px 선 금지(흐릿함 금지).
- **shadow**: blur 0 하드섀도만. 소형 요소 `4px 4px 0 var(--ink)`, 카드/버튼 `6px 6px 0`,
  모달/스테이지 `10px 10px 0`. **hover**: `translate(-2px,-2px)` + 섀도 `8px 8px 0`.
  **press**: `translate(6px,6px)` + 섀도 `none` (물리적으로 눌림). blur·glow·글래스 전면 금지.
- **스탬프 모티프**: WIN/LOSE/TOUCHÉ/BUST 등 판정 텍스트는 -8°~-15° 기울어진 보더 박스 스티커.
- **해저드 스트라이프**: `repeating-linear-gradient(45deg, #FFD600 0 12px, #0A0A0A 12px 24px)`.

### 1.4 모션 규칙

- **스냅 우선**: 트랜지션 120~180ms, `cubic-bezier(0.2, 0.9, 0.3, 1.15)` (살짝 오버슈트). 500ms 넘는 애니메이션 금지.
- **스탬프 등장**: `scale(1.6)→scale(1)` + 회전 안착, 200ms. 판정·결과에 사용.
- **화면 전환**: 페이드 금지 — 하드 컷 또는 검정 블록이 좌→우로 훑는 와이프(240ms).
- **키 입력 피드백**: 온스크린 키캡이 `translateY(4px)` + 섀도 소멸로 즉발 반응(트랜지션 0ms 다운 / 80ms 업).
- **카운트다운 임박(≤5초)**: 타이머 박스 배경이 `--accent`로 반전 + `steps(2)` 120ms 셰이크.
- **깜빡임은 steps()**: 대기 점(…), 매칭 스피너 대체 — 부드러운 회전 대신 90° step 회전.
- `prefers-reduced-motion`: 셰이크·스탬프 스케일 생략, 상태는 색/텍스트로만.

### 1.5 컴포넌트 스타일 지침

- **버튼**: Black Han Sans, 3px 보더 + 6px 하드섀도, hover/press 물리 규칙(§1.3).
  위계 — Primary(온라인 게임하기 등): `--accent` 배경 + ink 텍스트 / Secondary: `--surface` 배경 /
  Tertiary(로그아웃 등 텍스트성): 보더 없는 밑줄 2px. 파괴적(취소하기): `--surface` 배경 + `--error` 텍스트.
- **카드**: `--surface` + 3px 보더 + 6px 섀도. 제목은 카드 상단에 검정 스트립 바(ink 배경 + 크림 텍스트)
  또는 좌상단에 -3° 기울어진 라벨 스티커.
- **모달**: 오버레이 `rgba(10,10,10,0.55)`. 모달 본체는 4px 보더 + 10px 섀도 + 상단 검정 타이틀 바.
  타이틀 바 우측 끝에 장식 사각 3개(블루·핑크·옐로). 등장은 `scale(0.95)→1` 스냅 160ms.
- **입력**: `--surface` + 3px 보더, radius 0. focus 시 outline 대신 섀도 `4px 4px 0 var(--accent)`.
  에러 시 보더·섀도 `--error` + 아래 굵은 에러 텍스트. placeholder는 `--ink-muted`.
- **테이블(리더보드)**: 4px 외곽 보더, 헤더 행 ink 배경 + 크림 텍스트(Space Mono uppercase),
  행 디바이더 2px, 짝수행 `--surface-sunken`. 순위 1~3은 `--highlight` 배경의 기울어진 랭크 칩,
  내 행은 `--p1-tint` 배경 + 좌측 6px `--p1` 보더로 강조. 수치는 Space Mono 타뷸러 우측 정렬.
- **키캡(공통 부품)**: radius 8px, 3px 보더, 4px 섀도, Space Mono 대문자 키 문자 + 위에 화살표/아이콘.
  P1 키캡은 `--p1-tint`, P2 키캡은 `--p2-tint`. 실제 키 입력과 1:1로 눌림(§1.4).
- **공통 HUD 프레임(인게임 S9~S12)**: 상단 바에 좌(블루 프로필) ↔ 중앙(타이머+라운드) ↔ 우(핑크 프로필)의
  격투게임식 대칭 구조. 라운드 승수는 중앙 타이머 양옆에 사각 칩(획득=플레이어색, 미획득=빈 보더)으로
  중앙을 향해 배열. 세 게임이 이 프레임을 공유해 플랫폼이 한 시스템으로 묶인다.

### 1.6 구현 공통 사항 (기능 정본 준수)

- data-testid 레지스트리 전 항목 준수(scr-*, modal-*, btn-*, hud-*, lb-*, card-game*).
- dev 디버그 브리지 `window.__MADPUMP__ = { screen, game, session }` — 화면 전환·게임 틱마다 갱신.
- 게임 로직은 `@shared`의 `createGame1State/tick`, `createGame2State/tickGame2`,
  `createGame3State/tickGame3`, `computeLeaderboard`, 키맵(q/w, u/i)을 그대로 사용.

## §3. 게임 장면 아트 디렉션

### 3.1 게임1 — 숫자 펌핑: "세 개의 거대 숫자, 하나의 정답"
- **화면의 90%가 타이포.** 타겟(중앙 검정 박스) vs 좌우 플레이어 숫자(플레이어색 카드)의 3연 숫자 구도.
  숫자 변경 시 트윈 없이 **하드 스텝 교체** + 카드가 위(증가)/아래(감소)로 3px 튕김 — 연타의 타격감.
- **근접 피드백**: |현재−타겟| ≤ 5면 카드 보더가 `--accent`로, ≤ 2면 카드가 미세 셰이크 —
  "거의 다 왔다"의 긴장. (수치는 표시하지 않음 — 로직 비침범, 순수 연출.)
- **일치 유지 3초 = 스탬프 차징**: 일치 순간 플레이어 카드 위에 빈 보더의 "LOCKED" 스탬프가 나타나고
  내부가 플레이어색으로 3칸(1초 단위) 차오른다. 이탈 시 스탬프가 깨지며(2조각 분리 낙하) 리셋.
  3칸 완충 → 스탬프가 `--win` "WIN!"으로 쾅(스케일 스냅) + 상대 카드 회색 바램.
- 배경: 크림 위 초대형 아웃라인 "PUMP IT" 워터마크(5% 투명 ink) 대각 배치.

### 3.2 게임2 — 총알: "포탑 vs 러너, 캡슐의 비"
- **총알 = 알약 캡슐**: 흰 몸통 + 하단 `--accent` 팁, 3px 보더 + 2px 오프셋 미니 섀도.
  속도가 빠른 총알일수록 뒤에 2~3개의 고스트 잔상 사각(투명도 계단)을 끌고 내려온다 — blur 없는 속도감.
- **발사**: P1 포탑이 1스텝 반동(위로 4px) + 총구 위치에 사각 머즐 플래시 1프레임(옐로).
  쿨다운은 포탑 옆 탄창 칩이 빈 보더→채움으로 재장전되는 것으로 표시.
- **회피/피격**: P2 러너는 이동 방향으로 -6° 기울어 달리는 자세. 총알이 P2를 스치면(비충돌 근접)
  "CLOSE!" 미니 스티커가 그 좌표에 0.4초 팝. **피격 순간 화면 전체 1프레임 ink 반전** → P2 자리에
  사각 파편 4~6개가 하드섀도를 단 채 튀고, "HIT!" 스탬프(-12°, 에러색)가 쾅. P2 생존 승리 시
  하단 레인 전체가 핑크로 점등 + "SURVIVED!" 스탬프.
- 타이머 임박 5초: 중간 낙하 공간의 배경이 크림→`--highlight` 옅은 톤으로 상승, 긴박 강조.

### 3.3 게임3 — 펜싱: "일곱 칸 피스트와 지그재그 바다"
- **무대**: 옆면이 보이는 두꺼운 플랫폼 블록(상판 크림, 옆면 ink) — 7칸 경계마다 2px 눈금 + 칸 번호(Space Mono).
  양끝 바깥은 3단 지그재그 파도(진블루 `#2B5BFF` 계열 3톤 플랫 셰이프)가 steps()로 좌우 1스텝씩 출렁이고,
  플랫폼 끝단은 해저드 스트라이프 경계 — "여기 떨어지면 끝"이 즉독.
- **검객**: 기하 도형 조합(원 머리 + 사각 몸통 + 막대 팔다리)의 3px 아웃라인 캐릭터, 좌=블루/우=핑크 채움.
  공격=런지(앞으로 기울며 검 수평 돌출), 회피=상체 뒤로 + 방패 원호 전개, 무행동=검끝 겨눈 중립.
- **틱 판정 리듬**: 매 1초 틱마다 ①양측 머리 위 말풍선 칩에 선택 동시 공개(검/방패/… 아이콘) →
  ②판정 — 밀리는 쪽이 1칸을 **하드 스텝 슬라이드**(트윈 짧게 120ms)로 후퇴 + 발밑에 먼지 사각 2개 +
  "TOUCHÉ!" 스탬프(-15°)가 승자색으로 쾅. 비김이면 두 검이 중앙에서 교차하며 "CLASH" 미니 칩 + 4px 셰이크.
- **링아웃**: 캐릭터가 회전하며 바다로 낙하(2스텝 프레임) → 파도가 한 번 크게 지그재그 튀어오르고
  물기둥 사각 스플래시 → 승자 쪽 화면 절반이 승자색으로 하드 컷, "RING OUT!" 초대형 스탬프.
- 남은 칸 수는 각 플레이어 프로필 아래 사각 칩(3개)으로 병기 — 밀릴 때마다 하나가 깨져 회색화.
