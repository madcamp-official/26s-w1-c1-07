## 시각 이슈

### V-1. [실패] S2 리더보드 "내 등수" 행이 기준 뷰포트에서 하단 잘림

- 증거: `05-s2-main-in.png` / `06-s2-leaderboard.png` / `21-final-main-in.png` (잘린 상태) vs `22-s2-leaderboard-scrolled.png` (스크롤 후 정상)
- 측정 (probe-lb.mjs, 1280×832): lb-myrank rect top=806.75 / bottom=850.75, viewport 832 → **행 44px 중 약 19px가 폴드 밖**. 문서 높이 914px로 세로 스크롤은 정상 동작.
- 영향: SPEC 행24의 핵심 요소인 "내 등수"(9위 QA러너 0판 0승 0%)의 텍스트가 스크롤 없이는 읽히지 않음. S1은 한 화면에 완결되는 구성이라 S2도 스크롤 없는 로비로 읽히는데, 마지막 행만 살짝 넘쳐 미완성처럼 보임.
- 제안: S2 상하 여백 축소(`.s2-main` padding·`.s2-board` padding-top) 또는 리더보드 카드 행 높이 축소로 832px 내 수납. (또는 lb-myrank를 카드 상단에 고정)
- 심각도: 하 (기능은 동작, 스크롤로 열람 가능)

### 그 외 시각 점검 결과 (이상 없음)

- 폰트: 디스플레이/모노/바디 모두 적용됨 (미적용 폴백 없음)
- 대비: 본문·버튼·HUD 모두 충분. 모달 오픈 시 배경 딤 정상
- 겹침: "COPIED!" 스티커·네임태그의 요소 걸침은 의도된 스티커 연출로 판단 (가독성 훼손 없음)
- 결과 오버레이: stamp-in 연출(약 0.2s) 후 완전 불투명 — 최초 캡쳐가 애니메이션 중이었던 것으로 재확인 완료 (15·16·18·20·29·31)
- 게임1 인트로 "ROUND 1" 스탬프가 잠시 타겟 숫자를 가리나 0.9초 후 소멸 — 입력은 즉시 가능, 이슈 아님
- 스크롤: S2 외 화면 전부 단일 뷰포트 수납, 가로 스크롤 없음

## 스펙 일치성 메모 (실패 아님)

- 온라인 매칭은 봇 상대 배정 + 게임 랜덤(Q8 임시 처리 허용 범위), 오프라인은 PLAYER 1/2 로컬 2인 — SPEC §0.3 전제 준수
- 방 코드 11자리(Q9 와이어프레임 그대로), 코드 형식 검증은 숫자만(분반 제한 없음, S6-7)
- 라운드당 시간 기본 60초(Q1), 설정 모달 배경 클릭=저장 안 함(Q11), 중복 에러 후 입력 유지(Q12), 접속 중 단계 취소 버튼 없음(Q15) 모두 스펙 판정대로 구현됨
- 게임 로직은 @shared import 사용 확인 (`Game1/2/3.tsx` 모두 create*/tick* 호출, 재구현 없음)
