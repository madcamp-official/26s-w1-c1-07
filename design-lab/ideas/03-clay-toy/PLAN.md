## §1. 디자인 시스템 — "Clay Toy Battle"

컨셉 한 줄: **말랑한 찰흙 장난감끼리 싸우는 파스텔 놀이방.** 모든 UI가 무광 클레이 덩어리처럼
통통하게 부풀어 있고(이중 그림자 볼륨), 누르면 젤리처럼 짜부라진다. 경쟁의 긴장은 유지하되
스트레스는 뺀 '분노 없는 대결' 톤 — 루프 1(하드엣지 평면)·루프 2(다크 네온)와 정반대의
부드러운 볼륨감 축.

### 1.2 타이포 (구글 폰트 2종)

| 폰트 | 역할 |

### 1.3 형태 언어 — "빚어놓은 찰흙"

- **radius**: 기본 `24px`, 대형 카드·모달·스테이지 `32px`, 소형 배지·칩 `16px`, 키캡·아바타는
  완전 원/스쿼클. **날카로운 모서리 0개** — 4px 이하 radius 금지.
- **border**: 선(stroke) 없음이 기본. 윤곽은 그림자 볼륨으로만 만든다. 예외 — 선택/포커스 상태만
  `3px solid` 플레이어색 또는 `--accent` 링(바깥 offset 2px).
- **shadow (클레이 볼륨 공식)** — 모든 입체 요소에 3중 조합:
  ```css
  box-shadow:
    0 10px 24px rgba(74, 58, 82, 0.14),            /* 바깥 소프트 드롭 */
    inset 0 -10px 14px rgba(74, 58, 82, 0.12),     /* 아래 안쪽 음영 = 부푼 배 */
    inset 0 8px 10px rgba(255, 255, 255, 0.85);    /* 위 안쪽 하이라이트 = 무광 광택 */
  ```
  소형 요소는 수치 60%로 축소, 모달/스테이지는 바깥 드롭을 `0 18px 40px`로 확대.
- **눌린 홈(sunken)**: 입력창·게이지 트랙·테이블 배경은 반대로
  `inset 0 4px 8px rgba(74,58,82,.14), inset 0 -2px 4px rgba(255,255,255,.7)` — 찰흙을 손가락으로
  꾹 눌러 만든 자국.
- **press 상태**: 바깥 드롭 소멸 + `scale(0.96)` + 아래 inset 강화 — "말랑 눌림".
- **장식 모티프**: 화면 모서리에 크고 흐린 클레이 블롭(도넛·별·물방울, `--bg`보다 5% 진한 톤)을
  1~2개 깔아 놀이방 배경감. 정보 영역과 절대 겹치지 않게 z-index 최하단.

### 1.4 모션 규칙 — "젤리 물리"

- **기본 스프링**: `cubic-bezier(0.34, 1.56, 0.64, 1)` (백 오버슈트), 220~320ms.
- **젤리 프레스**: 버튼/키캡 press 시 `scale(0.94)` 즉발(0ms) → 떼면 `scale(1.03)→1` 스프링 260ms.
- **등장(모달·카드)**: `scale(0.7)+translateY(24px)` → 스프링으로 '뿅' 부풀며 안착. 페이드 단독 금지.
- **squash & stretch**: 승패·피격 등 임팩트 순간 대상에 `scaleX(1.15) scaleY(0.85)` → 복원 1회.
- **아이들 호흡**: 메인 타이틀·캐릭터는 `scale(1)↔(1.02)` 3s ease-in-out 무한 반복(살아있는 찰흙).
- **컨페티**: 승리 시 클레이 조각(원·별·초승달, 팔레트 5색)이 통통 튀며 낙하 — 회전 느리게, 탄성 바운스 1회.
- **카운트다운 임박(≤5초)**: 타이머 알약이 `--accent`로 물들며 심장박동 pulse(scale 1↔1.06, 500ms).
- **금지**: blur 글로우, 하드 컷 플래시(피격 플래시는 120ms 색 오버레이 1회만), 500ms 초과 블로킹 애니메이션.
- `prefers-reduced-motion`: 스프링→150ms ease-out 축소, 아이들 호흡·컨페티 정지.

### 1.5 컴포넌트 스타일 지침

- **버튼**: Jua, 클레이 볼륨 공식 적용된 통통한 알약/스쿼클.
  - Primary(온라인/오프라인 게임하기, 확인): `--accent` 배경 + `#FFF9F4` 텍스트.
  - Secondary(코드 생성, 복사, 기본값): `--surface` 배경 + `--ink` 텍스트.
  - Tertiary(로그아웃 등): 배경 없는 `--lavender` 텍스트 + hover 시 연보라 알약 배경.
  - 취소/파괴(취소하기): `--surface` 배경 + `--error` 텍스트.
  - Google 로그인: 흰 클레이 알약 + 공식 G 로고 + "SIGN IN WITH GOOGLE".
- **카드**: `--surface` + radius 32 + 클레이 볼륨. 제목은 Jua, 카드 상단에 작은 클레이 아이콘
  오브젝트(공·별 등)를 얹어 장난감 상자 느낌.
- **모달**: 딤은 `rgba(74,58,82,0.35)` + 살짝 라일락. 모달 본체는 카드와 동일하되 바깥 드롭 확대.
  등장은 §1.4 '뿅' 스프링. 배경 클릭 닫기 지원(SPEC 관례 따름).
- **입력**: sunken 홈 스타일(§1.3) + radius 20. 포커스 시 `--accent` 3px 링. 에러 시 `--error` 링 +
  아래 에러 텍스트(시스템 폰트 14px) + 좌우 4px 흔들림 1회.
- **테이블(리더보드)**: 괘선 없는 클레이 판. 행 = 둥근 트레이(짝수행 `--surface-sunken`).
  순위 1~3위는 금·은·동 클레이 메달 원판, 숫자는 Baloo 2 tabular. 내 행은 `--accent-soft` 트레이 +
  "나" 배지. 빈 상태는 납작하게 눌린 찰흙 얼굴 + "기록 없음" (가짜 데이터 금지 — SPEC §0.4).
- **키캡(온스크린 패드)**: 완전 원형 클레이 버튼, 위에 화살표/아이콘, 아래 Baloo 2로 실제 배정 키
  (Q/W/U/I) 각인. 플레이어 소속색 tint 배경. 실제 키 입력과 1:1로 젤리 프레스.
- **토스트**: 하단 중앙 알약("코드 복사됨!"), 뿅 등장 → 2초 후 쪼그라들며 퇴장.

## §2. 페이지별 계획 (S1~S12)

공통 셸: 배경 `--bg` + 모서리 클레이 블롭 장식. 화면 전환은 젤리 스프링 등장(라우트 컨테이너).
data-testid 레지스트리·`window.__MADPUMP__` 브리지는 전 시안 공통 규약대로 구현.

### S1. 메인 — 비로그인 (`scr-main-out`)
- 기능(SPEC): MADPUMP 타이틀 / SIGN IN WITH GOOGLE / 설정 버튼 / 온라인·오프라인 게임하기 / 비로그인 온라인→S3 모달.
- 표현: 중앙에 초대형 Jua "MADPUMP" — 글자 하나하나를 클레이 덩어리 조각처럼 색 교차(coral/mint/butter/lavender)
  + 아이들 호흡. 타이틀 아래 세로 스택으로 Primary 버튼 2개(`btn-online`, `btn-offline`).
  우상단 헤더: Google 로그인 알약(`btn-google-login`) + 원형 톱니 클레이 버튼(`btn-settings`).
  타이틀 주변에 게임 3종을 암시하는 미니 클레이 오브젝트(풍선·총알캡슐·검) 둥둥 부유.
### S2. 메인 — 로그인 후 (`scr-main-in`)
- 기능: 닉네임 인사말 / 로그아웃 / 분반 리더보드(TOP3 + 내 등수, 플레이수·승수·승률) / 온·오프라인 진입 / 설정.
- 표현: S1 레이아웃 유지하되 타이틀을 좌측으로 옮기고 우측에 리더보드 클레이 트레이 패널
  (제목 "○분반 리더보드", `lb-top3`=메달 3행, `lb-myrank`=하단 고정 내 행). 우상단 헤더에
  "○○님 안녕하세요"(Jua) + 로그아웃 Tertiary + 설정. 빈 데이터면 납작 찰흙 얼굴 빈 상태.
### S3. 로그인 요구 모달 (`modal-login-required`)
- 기능: 안내 문구 / 모달 내 Google 로그인(성공 시 S6 연속 진입) / 취소하기 / 배경클릭·ESC 취소.
- 표현: 소형 클레이 카드가 '뿅'. 상단에 자물쇠 모양 라벤더 클레이 오브젝트, 문구
  "온라인 게임은 로그인이 필요합니다!" (Jua), 아래 Google 알약 + 취소 버튼 세로 스택.
### S4. 설정 모달 (`modal-settings`)
- 기능: 라운드 수(round) / 라운드 당 시간(초) / 확인(`btn-settings-save`) / 기본값 / 배경클릭=저장 안 함. 기본 3라운드·60초.
- 표현: "설정" 제목 + 톱니 클레이 아이콘. 숫자 입력은 sunken 홈 + 좌우에 −/＋ 미니 클레이 버튼
  (스텝퍼, 키보드 입력도 허용) + 단위 라벨. 하단 확인(Primary)/기본값(Secondary) 가로 배치.
### S5. 닉네임 온보딩 (`scr-onboarding`)
- 기능: "What's your name?" / 이름·분반 입력(`input-nickname`) / 확인(`btn-nickname-submit`) / 중복 에러(`err-nickname-dup`) / 빈값 제출 방지 / 에러 후 수정 시 해제.
- 표현: 화면 중앙 명찰 모양 대형 클레이 카드 — 유치원 이름표 메타포. 제목은 Baloo 2+Jua 혼용,
  카드 위에 이름표 끈처럼 보이는 클레이 고리 장식. "test" 제출 시 카드가 좌우 흔들림 + 빨간 에러 텍스트.
  성공 시 카드가 젤리 바운스 후 S2로.
### S6. 온라인 게임하기 패널 (`modal-online`)
- 기능: 빠른 시작(`btn-quickstart`) / 코드 생성(`btn-code-create`)→코드 표시(`room-code-display`)+복사+톱니(S4 재사용) / 코드 입력(`input-code`)+확인(`btn-code-join`) / 배경 클릭 닫기 / 코드는 분반 제한 없음.
- 표현: `--bg-sky` 톤의 대형 클레이 패널, 제목 "온라인 게임하기". 상단에 풀폭 빠른 시작 Primary
  (번개 클레이 아이콘). 아래 "게임 만들기 / 참가하기" sunken 구역: 1행 코드 생성 버튼 + 코드가
  구슬 목걸이처럼 한 자리씩 또르르 등장하는 표시부 + 복사 버튼(토스트 피드백) + 미니 톱니.
  2행 코드 입력 홈 + 확인. 잘못된 형식은 에러 링+흔들림.
### S7. 온라인 매칭 상태 모달 (`modal-matching`)
- 기능: connecting("게임에 접속 중입니다")→waiting("플레이어 대기 중"+취소 `btn-matching-cancel`) / 성사 시 인게임 자동 전환 / 취소 시 타이머 정리 후 S6 복귀.
- 표현: 소형 카드 중앙에 클레이 공 3개(coral/butter/mint)가 차례로 통통 튀는 로더.
  waiting 단계에서 문구 교체 + 취소 버튼 등장. 매칭 성사 순간 P1/P2 색 클레이 인형 2개가
  맞부딪히며 "매칭 완료!" 짧은 연출(≤600ms) 후 게임으로.
### S8. 게임 선택 (`scr-game-select`)

## §3. 게임 장면 아트 디렉션

### 3.1 게임1 — 숫자 펌핑: "풍선 부풀리기 공작 시간"
- 무대: `--bg` 피치 크림 위 중앙에 거대한 타겟 숫자 클레이 판(스쿼클, `--pop` 버터 옐로 테두리 톤) —
  Baloo 2 ExtraBold 목표 숫자가 도장 찍은 찰흙처럼 음각.
- 양측 현재 숫자 = **클레이 풍선**: P1 딸기핑크 / P2 민트 풍선에 흰 숫자. 값이 오르면 풍선이
  `scale` 스프링으로 한 번 부풀고(w/i), 내리면 살짝 쪼그라든다(q/u). 타겟과의 거리 비례로
  풍선 크기 미세 매핑 — 가까울수록 통통해진다.
- 일치 유지 3초: 풍선 둘레에 클레이 도넛 링이 3칸으로 차오르는 홀드 게이지 + 풍선이 반짝
  하이라이트. 이탈 시 링이 툭 떨어져 부서진다(리셋의 정직한 표현).
- 승리: 이긴 풍선이 별 모양으로 팡 터지며 클레이 컨페티, 진 풍선은 바람 빠져 쪼글쪼글 —
  귀엽게 삐진 얼굴이 그려진다.
- 패드: 하단 좌우에 ↓/↑ 원형 클레이 키캡(Q·W / U·I 각인, 소속색 tint). "this is you"는
  내 풍선 위 말랑 화살표 태그 "나!"로 번역.

### 3.2 게임2 — 총알 피하기: "장난감 구슬 비"
- 무대: `--bg-sky` 하늘 파스텔 스테이지(radius 32 클레이 프레임). 상단 트랙 = 구름 모양 클레이
  레일 위를 미끄러지는 P1 인형(딸기핑크, 대포 장난감을 안고 좌우 왕복). 하단 트랙 = 잔디색
  아닌 `--surface-sunken` 선반 위 P2 인형(민트).
- 총알 = 알약 모양 클레이 캡슐(라벤더+흰색). 발사 순간 P1 인형이 squash&stretch로 반동,
  캡슐은 낙하 중 살짝 출렁인다(속도 랜덤은 물리 그대로, 잔상·글로우 없음).
- 장전/쿨다운: P1 머리 위 반투명 캡슐 실루엣이 차오르는 클레이 게이지 — 채워지면 통통 튀며 활성.
- 피격: P2 인형이 납작하게 짜부라지고(찰흙 팬케이크) `--error` 오버레이 120ms 1회.
  이어 결과 오버레이. 생존 승리: P2 인형이 브이 포즈 젤리 바운스, 하늘에서 버터옐로 별 컨페티.
- 방향 반전(q): P1 인형이 몸을 홱 돌리는 뒤집기 애니메이션으로 즉시 피드백.
  패드: 중앙 하단 반투명 클레이 키캡 — P1측 ⟲/발사, P2측 ←/→에 실제 키 각인.

### 3.3 게임3 — 펜싱: "찰흙 해적판 위의 결투"
- 무대: 화면 하단 1/3이 `--sea` 클레이 바다 — 물결이 느리게 출렁이는 둥근 파도 조각들.
  그 위에 떠 있는 **칸이 새겨진 클레이 판자 다리**(7칸: 각 3칸+중앙, 칸 경계는 눌린 홈 선) —
  좌우 끝은 둥글게 부서진 낭떠러지 단면. 배경은 `--bg-lilac` 노을 파스텔.
- 캐릭터: P1 딸기핑크 / P2 민트 클레이 인형 — 이쑤시개 검과 단추 방패를 든 통통한 눈사람 체형.
  스탠스별 포즈: 공격=검 내지르기 런지, 회피=방패 웅크림, 무행동=검 든 채 아이들 호흡.
- 1초 틱 연출: 틱마다 무대 상단 중앙에 양측 선택이 클레이 아이콘 칩(검/방패/…)으로 '뿅' 공개 →
  판정. 밀리는 쪽 인형이 squash로 밀려나 옆 칸에 착지(스프링 안착), 칸 홈에 발자국 자국.
- 낭떠러지 경고: 남은 1칸이 되면 그 쪽 판자 끝이 삐걱 기울고 물결이 높아진다.
- 링아웃: 인형이 풍덩 — `--sea` 물기둥 클레이 스플래시 + 물 위로 삐진 얼굴만 동동. 승자는
  판자 위에서 검 들고 젤리 바운스 세리머니, 컨페티 낙하.
- 카운트다운: 무대 상단 알약 타이머(공통) + 틱 리듬에 맞춰 작은 메트로놈 점이 좌우로 통통.
- 패드: 좌하/우하에 검(공격)·방패(회피) 원형 클레이 키캡, 실제 키(Q·W / U·I) 각인. 틱 내
  마지막 입력이 칩으로 미리보기 표시(내 쪽만).
