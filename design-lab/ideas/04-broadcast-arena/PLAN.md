## §1. 디자인 시스템

### 1.1 팔레트 (정확한 hex)

| 역할 | 토큰 | hex | 용도 |
|---|---|---|---|
| 배경(주) | `--bg` | `#EEF2F7` | 스튜디오 라이트 그레이블루. 전 화면 기본 배경 |
| 배경(필드) | `--bg-field` | `#E3EAF3` | 게임 스테이지·패널 뒤 미세 그라디언트 짝 |

| 앵커 스트립 | `--strip` | `#0E1E3C` | 중계 네이비 — 하단 티커 바·primary 버튼·스코어 버그 몸체 |
| 텍스트(주) | `--ink` | `#0E1E3C` | 네이비 잉크. 흰 표면 위 본문/헤드라인 |

| 포인트(골드) | `--gold` | `#C99312` | 트로피 골드 — 승리·1위·MVP **전용** (밝은 짝 `#E4B33C` 그라디언트) |

- **P1 vs P2 색 구분은 SPEC 필수(주석 16:1713)**: 인게임 전 화면에서 P1=블루 / P2=레드를
  배지·게이지·캐릭터·패널 엣지에 일관 적용. 내 쪽은 추가로 `YOU` 태그 + 팀 컬러 4px 엣지로 강조.
- 골드는 승부의 순간(승리 연출, 1위 행, MVP 카드)에만 허용 — 평시 화면에 골드 금지.

### 1.2 타이포 (구글 폰트 2종 — 한글 지원 1종 포함)

| 폰트 | 용도 |
|---|---|
| **Archivo** (variable: wdth 62–125, ital) | 방송 디스플레이 — 익스텐디드 볼드 대문자 헤드라인(`wdth 125`), 컨덴스드 라벨(`wdth 70`), 이탤릭 기울임으로 속도감. 숫자는 `font-feature-settings:"tnum"` 타뷸러 고정 |
| **IBM Plex Sans KR** | 한글 본문·버튼 텍스트·UI 전반 (400/500/700) |

`index.html`에 넣을 link 태그:

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Archivo:ital,wdth,wght@0,62..125,100..900;1,62..125,100..900&family=IBM+Plex+Sans+KR:wght@400;500;700&display=swap" rel="stylesheet">
```

- 스케일: 워드마크 96–128px(이탤릭 스큐), 스코어 버그 숫자 40px tnum, 섹션 라벨 12px
  대문자 letter-spacing 0.12em, 본문 15px, 캡션 12px.

### 1.3 형태 언어 (radius / border / shadow)

- **스큐 -12° 평행사변형**이 시그니처: 로워서드·섹션 헤더 탭·primary 버튼은
  `transform: skewX(-12deg)` (내부 텍스트는 역스큐로 수평 유지).
- radius: 스큐 요소 0px / 일반 카드·모달 6px / pill(스코어 버그·LIVE 뱃지) 999px.
- border: 1px `--line` 기본. 카드에는 **상단 또는 좌측 4px 컬러 바**(팀 컬러·골드·네이비)로 소속 표시.
- shadow: `0 8px 24px rgba(14,30,60,.12)` 단일 소프트 섀도(방송 패널 부유감). 하드 섀도 금지.
- 글래스: 게임 스테이지 **위에 얹히는** HUD만 `rgba(255,255,255,.82)` + `backdrop-filter: blur(12px)`.
- 하단 티커: 화면 최하단 풀폭 네이비 스트립(높이 36px) — 전 화면 공통 앵커.

### 1.4 모션 규칙

- 기본 이징 `cubic-bezier(.2,.8,.2,1)`, 기본 시간 240ms.
- **로워서드 와이프 인**: 좌→우 clip-path 와이프 + 스큐 유지 (패널·모달 등장 공통).
- **플립보드 카운터**: 점수·타겟 숫자 변경 시 세로 플립/롤 120ms.
- **LIVE 점**: 1s 주기 opacity 펄스. **티커**: 30s 무한 linear 스크롤.
- **승부 순간**: 120ms 화이트 플래시 와이프 → 0.5s 슬로모션(scale .98→1) → 골드 파티클 + result 오버레이.
- 카운트다운 마지막 5초: 숫자 `--live` 레드 전환 + 스케일 펄스.
- `prefers-reduced-motion: reduce` 시 와이프·파티클·티커 정지(정적 표시로 대체).

### 1.5 컴포넌트 스타일 지침

- **버튼**: primary = 네이비 스큐 블록 + 흰 대문자(Archivo cond) — hover 시 골드 언더라인이
  좌→우 와이프, active 시 1px 하강. secondary = 흰 바탕 1px `--line` 보더 스큐 블록.
  Google 로그인 버튼만 관례 유지(흰 바탕 + 컬러 G 로고 + "SIGN IN WITH GOOGLE", 스큐 없음).
- **카드**: 흰 표면 + 1px 보더 + 4px 컬러 바 + 소프트 섀도. 헤더는 스큐 탭 라벨(대문자 12px).
- **모달**: 딤 `rgba(14,30,60,.45)`. 카드 상단에 스큐 헤더 탭(예: `MATCH RULES`, `SIGN-IN REQUIRED`)
  — 탭 색으로 성격 구분(안내=네이비, 경고=LIVE 레드). 등장은 로워서드 와이프.
- **입력**: 흰 바탕 1px 보더 radius 4px, 포커스 시 2px 네이비 아웃라인 + 좌측 3px 팀/네이비 스트립.
  코드·숫자 입력은 tnum 고정폭. 에러는 `--live` 레드 1px 보더 + 하단 레드 캡션.
- **테이블(리더보드)**: 방송 순위표 — 좌측 순위 번호 네이비 블록(1위만 골드), 1px 로우 디바이더,
  열: 순위/닉네임/경기·승·승률(tnum 우측 정렬). 내 행은 팀 블루 틴트 하이라이트 + `YOU` 태그.

### 1.6 구현 공통 (전 화면)

- 게임 로직은 `@shared`(`@madpump/shared`)의 `createGame1State/tick` 등 **import 전용, 재구현 금지**.
  키맵 playerL q/w, playerR u/i. mock 데이터·`computeLeaderboard`도 shared 사용.
- data-testid 레지스트리 전수 준수 (scr-*, modal-*, btn-*, hud-*, lb-*, card-game1~3 등).
- dev 디버그 브리지: `window.__MADPUMP__ = { screen, game, session }` — 화면 전환·게임 틱마다 갱신.
- 리더보드 등 mock 부재 자리는 빈 상태를 정직하게 표기(SPEC §0.4).

## §2. 페이지별 계획 (S1~S12)

### S1. 메인 — 비로그인 (`scr-main-out`)
- SPEC 기능: MADPUMP 타이틀 / SIGN IN WITH GOOGLE(`btn-google-login`) / 설정(`btn-settings`) /
  온라인(`btn-online`)·오프라인(`btn-offline`) 진입, 비로그인 온라인 → S3 모달.
- 표현: **프리게임 스튜디오**. 상단 얇은 방송 헤더 바(좌: 스큐 채널 로고 "MP ARENA", 우: Google
  버튼 + 원형 설정 버튼). 중앙에 초대형 이탤릭 익스텐디드 워드마크 `MADPUMP` + 서브라벨
  "OFFICIAL PUMPING LEAGUE — 26S W1". 그 아래 두 버튼 세로 스택: 온라인=네이비 primary
  ("온라인 게임하기 · RANKED"), 오프라인=secondary ("오프라인 게임하기 · EXHIBITION").
  하단 풀폭 네이비 티커: mock 최근 매치 결과·안내 문구가 스크롤. 배경엔 미세한 피치 라인 패턴.

### S2. 메인 — 로그인 후 (`scr-main-in`)
- SPEC 기능: 닉네임 인사말 / 로그아웃 / 분반 리더보드(TOP3 스탯 + 내 등수) / 온라인·오프라인 / 설정.
- 표현: 헤더 우측에 **로워서드 인사말**("OOO님 안녕하세요" — 스큐 네임플레이트, 팀 블루 엣지) +
  로그아웃 텍스트 버튼 + 설정. 워드마크는 좌측으로 시프트, 우측에 **STANDINGS 패널**:
  스큐 헤더 탭 "STANDINGS — 1분반", TOP3 순위표(`lb-top3`; 1위 골드 블록, 경기/승/승률 tnum),
  디바이더 아래 내 등수 행(`lb-myrank`, 블루 틴트 + YOU). mock 없으면 "기록 없음" 빈 상태.
  하단 버튼 2개·티커는 S1과 동일 문법.

### S3. 로그인 요구 모달 (`modal-login-required`)
- SPEC 기능: 가드 문구 / 모달 내 Google 로그인 → 성공 시 S6 연속 진입 / 취소 → S1.
- 표현: **BREAKING 스타일 알림 카드** — 상단 스큐 탭이 LIVE 레드 "SIGN-IN REQUIRED",
  본문 "온라인 게임은 로그인이 필요합니다!", Google 버튼 + secondary "취소하기".
  ESC/배경 클릭 = 취소.

### S4. 설정 모달 (`modal-settings`)
- SPEC 기능: 라운드 수 / 라운드 당 시간(단위 round·초) / 확인(`btn-settings-save`) / 기본값.

- 표현: 스큐 탭 "MATCH RULES" 네이비. 두 입력을 **경기 규정 카드** 2행으로: 라벨(대문자
  "ROUNDS", "TIME / ROUND") + 숫자 입력(tnum, min 1) + 단위. 하단 우측 확인(primary)/기본값(secondary).

### S5. 닉네임 온보딩 (`scr-onboarding`)

### S6. 온라인 게임하기 패널 (`modal-online`)
- SPEC 기능: 빠른 시작(`btn-quickstart`) / 코드 생성(`btn-code-create`) + 코드 표시

- 표현: **MATCHMAKING 대형 패널** — 스큐 탭 "MATCHMAKING". 상단에 풀폭 빠른 시작 primary
  ("빠른 시작 · QUICK MATCH"). 아래 "게임 만들기 / 참가하기" 섹션(1px 보더 서브카드):
  1행 코드 생성 버튼 + 코드 표시 슬롯(생성 전 밑줄 플레이스홀더 → 생성 후 tnum 코드) +
  복사 버튼("복사됨" 토스트) + 톱니 아이콘. 2행 코드 입력 + 확인. 코드 미생성 시 복사 비활성.

### S7. 온라인 매칭 상태 모달 (`modal-matching`)

  (신호 바 4개가 순차 점등). waiting 단계에서 "플레이어 대기 중" + 상대 슬롯이 빈 네임플레이트로
  깜박임, 취소하기 버튼 노출(원안 충실: waiting에만, Q15). 성사 순간 두 네임플레이트가

  P1승/P2승/무승부) + MVP 스탯 라인 + `btn-next-round` / `btn-back-main`. 승자 쪽 골드 파티클.
- 설정 라운드 수만큼 반복, playerL/R → P1/P2 코인토스 mock, 색상 구분 필수.

### S9. 게임1 — 숫자 맞추기 (`scr-game1`)
- SPEC 기능: 타겟 숫자(1~100) / 양측 현재숫자 + 내 쪽 구분 / q↓ w↑·u↓ i↑ / 일치 3초 유지 승리 /
  배정≠타겟 / 카운트다운 / 프로필 / 패드에 화살표+실제 키 표기.
- 표현: §3.1 참조. 중앙 타겟 플립보드, 좌우 팀 컬러 현재숫자 패널, 유지 타이머 링.

### S10·S11. 게임2 — 총알 피하기 (`scr-game2`)
- SPEC 기능: 상단 P1 트랙(자동 왕복, q=방향 반전, w=발사) / 하단 P2 트랙(u←, i→) / 총알 낙하
  속도 랜덤 / 쿨다운(장전) 표시 / 피격=P1 승, 생존=P2 승 / 카운트다운 양 시점 표시 / 색상 구분.
- 표현: §3.2 참조. 로컬 2인은 한 화면(상하 트랙 동시 노출), P1/P2 "시점"은 내 배지
  하이라이트(YOU) 차이로만 구현.

### S12. 게임3 — 펜싱 (`scr-game3`)
- SPEC 기능: 1초 틱 가위바위보(공격 q·u/회피 w·i/무행동, 마지막 입력 채택) / 상성 판정·1칸 밀림 /
  시작 3칸 / 링아웃(바다) / 시간 종료 시 더 밀린 쪽 패배 / 검·방패 포즈 시각화 / 카운트다운 / 색상 구분.
- 표현: §3.3 참조. 피스트(스트립) 무대 + 바다, 틱 판정 배너.

## §3. 게임 장면 아트 디렉션

### 3.1 게임1 — 숫자 펌핑: "스플릿스크린 기록 경기"
- 스테이지를 세로 1px 헤어라인으로 이분할한 **기록실 듀오톤**: 좌측 필드에 블루 틴트
  `#E3EEFD`, 우측에 레드 틴트 `#FCE8E9`가 아주 옅게 깔린다.
- 중앙 상단에 **타겟 숫자 플립보드**: 공항 출발판/스타디움 전광판풍 검은 슬랫에 흰 숫자,
  "TARGET" 스큐 라벨. 라운드 시작 시 슬랫이 촤르륵 돌아가며 타겟 공개.
- 좌우 하단 **현재숫자 패널**: 팀 컬러 4px 엣지 + 40px tnum 숫자, 변경 시 세로 롤 애니메이션.
  내 패널엔 YOU 태그. 타겟과의 차이가 8 이내로 좁혀지면 패널 보더가 팀 컬러로 발광(글로우는
  승부 근접 순간에만 — 모션 규칙 준수).
- **일치 유지 3초**: 일치 순간 해당 패널 둘레에 골드 프로그레스 링이 차오르고 "HOLD" 라벨 점멸,
  중계 카메라처럼 스테이지 전체가 살짝 줌 인. 이탈 시 링이 깨지며 회색으로 리셋.
- 하단 패드: ↓/↑ 화살표 원형 키캡에 실제 키(q/w, u/i) 각인, 입력마다 팀 컬러 점등.
- 승리 판정: 화이트 플래시 → 승자 숫자가 골드로 전환 + "MATCHED!" 로워서드 와이프.

### 3.2 게임2 — 총알: "듀오톤 동시 중계 + 리플레이"
- 필드는 밝은 `--bg-field` 위 **상하 두 트랙**: 상단 P1 트랙은 블루 헤어라인 + 블루 배지,
  하단 P2 트랙은 레드 헤어라인 + 레드 배지. 배지는 원형 네임플레이트(이니셜).
- **총알 캡슐**: 네이비 몸체 + 발사 직후 팀 블루 모션 트레일(속도감 이탤릭 잔상 3개).
  낙하 속도 랜덤(0.8~2.0s)이라 트레일 길이도 속도에 비례 — 빠른 탄일수록 길게.
- **장전/쿨다운**: P1 배지 옆 반투명 캡슐 실루엣이 0.5초 동안 차오르는 게이지로 재장전 표현.
- **아슬 회피**: 총알이 P2 반경 근접 통과 시 노란(골드 톤 아님, `#E4B33C` 80%) 서클
  하이라이트 1프레임 + "CLOSE!" 마이크로 라벨 — 관전 재미 포인트.
- **피격 순간**: 화이트 플래시 와이프 → 0.5s 슬로모션으로 마지막 탄도 재생(리플레이 프레이밍,
  화면 모서리에 "REPLAY" 스큐 태그) → result 오버레이에 MVP 스탯 카드(생존 시간·회피 횟수).
- 하단 키 인디케이터: P1 쪽 q(방향 반전)·w(발사), P2 쪽 u(←)·i(→) 키캡 점등.

### 3.3 게임3 — 펜싱: "피스트 스트립 + 바다 링아웃"
- 무대는 **펜싱 피스트(스트립)를 측면에서 본 방송 그래픽**: 밝은 하늘 그라디언트
  (`#EEF2F7`→`#E3EAF3`) 아래 흰 플랫폼, 칸 경계는 1px 네이비 헤어라인 + 칸 번호 스텐실.
  좌우 끝은 플랫폼이 끊기고 **바다**: 옅은 블루 웨이브 2겹이 느리게 루프(패럴랙스), 경계에
  레드/화이트 해저드 에지 라인.
- 캐릭터는 **선 굵은 스틱 펜서 2명**(P1 블루/P2 레드, 4px 스트로크): 공격=검 찌르기 런지 포즈,
  회피=방패 웅크림 포즈, 무행동=중립 스탠스. 포즈 전환 120ms 스냅.
- **1초 틱 판정**: 매 틱 중앙 상단에 판정 배너가 로워서드 와이프로 등장 — 양측 선택 아이콘
  (⚔/🛡/–를 픽토그램으로) + 결과("P1 PUSHES!"). 밀리는 쪽은 1칸 뒤로 미끄러지며 발밑에
  스키드 라인, 스코어 버그의 칸 카운터가 플립.
- **링아웃**: 마지막 칸에서 밀리면 슬로모션 낙하 → 물기둥 스플래시(팀 컬러 물방울) →
  화이트 플래시 → result 오버레이 "RING OUT!". 시간 종료 판정은 배너로 남은 칸 비교 공개.
- 카운트다운은 스코어 버그 중앙, 마지막 5초 레드 펄스. 키 인디케이터: q/u=공격, w/i=회피.
