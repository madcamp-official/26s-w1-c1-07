# 16:1264 — 메인 - 로그인 되기 이전

- 이미지: `design-lab/reference/figma/16-1264__메인_-_로그인_되기_이전.png`
- 캔버스 위치: (-5163, -167) — ver4 플로우의 시작점(최상단 열)

## 역할 추정
서비스 진입 시 처음 보는 **메인(랜딩) 화면의 비로그인 상태**. 유저 플로우의 출발점. 여기서 온라인/오프라인 게임 분기와 로그인이 시작된다. 로그인 후 상태는 별도 프레임(16:1474 "메인 - 로그인 한 뒤")이 존재한다.

## 구성요소 (위치 포함)
- **우상단**: "SIGN IN WITH GOOGLE" 버튼 (구글 G 로고 + 텍스트, 구글 표준 로그인 버튼 형태)
- **최우상단 모서리**: 원형 버튼 "설정"
- **중앙 상단**: 대형 타이틀 텍스트 "MADPUMP"
- **중앙 하단 (세로 스택 2개 버튼)**:
  - "온라인 게임하기" (회색 사각 버튼)
  - "오프라인 게임하기" (회색 사각 버튼)
- **오프라인 버튼 우측 하단에 걸친 톱니바퀴(설정) 아이콘** — 우상단 "설정" 원형 버튼과 같은 기능을 가리키는 주석성 아이콘으로 보임 (와이어프레임 특성상 아이콘 배치 실험 흔적일 가능성)

## 기능 목록
1. **구글 로그인**
   - (a) Google OAuth로 계정 로그인. 로그인하면 메인이 "로그인 한 뒤" 상태(16:1474)로 전환.
   - (b) 트리거: 우상단 "SIGN IN WITH GOOGLE" 클릭.
   - (c) mock: 클릭 시 가짜 유저(닉네임/아바타)를 localStorage에 저장하고 로그인 상태 UI로 전환. 실제 OAuth 생략.
2. **온라인 게임하기 진입**
   - (a) 온라인 매칭/방 만들기 패널(16:1321) 열기. 단, 비로그인 상태에서는 "로그인이 필요합니다" 모달(16:1276)을 먼저 띄움.
   - (b) 트리거: "온라인 게임하기" 버튼 클릭.
   - (c) mock: 로그인 플래그 검사 → 없으면 로그인 요구 모달, 있으면 온라인 패널 표시.
3. **오프라인 게임하기 진입**
   - (a) 로컬(같은 기기) 대결 모드로 진입. 게임 선택 화면(23:2529의 게임1/2/3 선택으로 추정)으로 이동.
   - (b) 트리거: "오프라인 게임하기" 버튼 클릭.
   - (c) mock: 라우팅만으로 구현 가능. 로그인 불필요.
4. **설정 열기**
   - (a) 라운드 수/라운드 당 시간 설정 모달(16:1293 참고) 열기.
   - (b) 트리거: 우상단 "설정" 원형 버튼 (또는 톱니 아이콘) 클릭.
   - (c) mock: 모달 오픈, 값은 localStorage 저장.

## 특이사항
- 같은 이름의 프레임이 2개 존재: 16:1264(이 프레임)와 23:2529. **23:2529는 로그인 버튼/설정/온·오프라인 버튼이 없고 "게임1/게임2/게임3" 선택 버튼만 있는 완전히 다른 레이아웃** — 이전 버전 초안 또는 게임 선택 단계 스케치로 추정.
- 톱니 아이콘이 오프라인 버튼과 겹쳐 그려져 있음 — 최종 구현 시 설정 진입점은 우상단 하나로 통일하는 게 자연스러움.
