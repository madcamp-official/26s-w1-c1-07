# 16:1276 — 온라인 게임하기 - 로그인이 안되어있을때 (로그인 요구 모달)

- 이미지: `design-lab/reference/figma/16-1276__온라인_게임하기_-_로그인이_안되어있을때.png`
- 캔버스 위치: (-5163, 3807)

## 역할 추정
비로그인 상태에서 메인의 "온라인 게임하기"를 눌렀을 때 뜨는 **로그인 요구 모달**. 배경에 메인 화면(16:1264)이 그대로 깔려 있고 그 위에 모달이 덮인 구조. 유저 플로우: 메인(비로그인) → 온라인 게임하기 클릭 → 이 모달 → 로그인 성공 시 온라인 패널(16:1321)로.

## 구성요소 (위치 포함)
- **배경 (메인 화면 그대로)**: 우상단 "SIGN IN WITH GOOGLE", 원형 "설정" 버튼, 가려진 "MADPUMP" 타이틀, "온라인 게임하기"/"오프라인 게임하기" 버튼, 톱니 아이콘
- **중앙 모달 (회색 박스)**:
  - 안내 문구: "온라인 게임은 로그인이 필요합니다!"
  - "SIGN IN WITH GOOGLE" 버튼 (구글 로고 포함, 모달 내 중앙)
  - "취소하기" 버튼 (진회색, 로그인 버튼 바로 아래)

## 기능 목록
1. **모달 내 구글 로그인**
   - (a) 로그인 수행 후 원래 의도했던 온라인 게임 패널로 이어서 진입.
   - (b) 트리거: 모달의 "SIGN IN WITH GOOGLE" 클릭.
   - (c) mock: 가짜 로그인 처리 → 모달 닫고 온라인 게임하기 패널(16:1321) 오픈.
2. **취소**
   - (a) 모달 닫고 메인 화면으로 복귀.
   - (b) 트리거: "취소하기" 클릭 (배경 클릭/ESC도 관례상 동일 동작 권장).
   - (c) mock: 모달 언마운트.
3. **(전제) 로그인 가드**
   - (a) 온라인 진입 시 로그인 여부 검사하는 로직 자체.
   - (b) 트리거: 메인의 "온라인 게임하기" 클릭 시점.
   - (c) mock: `if (!user) showLoginModal()`.

## 특이사항
- 같은 이름 프레임이 2개: 16:1276(이 프레임, 로그인 요구 모달)과 16:1293(설정 모달). **16:1293은 이름과 달리 내용이 설정 모달**이므로 혼동 주의.
- 모달 뒤 배경이 딤 처리 없이 그대로 보임 — 와이어프레임이므로 구현 시 딤/오버레이 여부는 자유.
