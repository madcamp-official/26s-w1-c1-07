# 16:1321 — 온라인 게임하기 (코드 생성 전)

- 이미지: `design-lab/reference/figma/16-1321__온라인_게임하기.png`
- 캔버스 위치: (-5163, 4916)

## 역할 추정
로그인 후 "온라인 게임하기"를 눌렀을 때 열리는 **온라인 매칭/방 관리 패널**의 초기 상태(방 코드 아직 미생성). 메인 화면 위에 큰 패널이 덮인 구조. 유저 플로우: 메인 → (로그인) → 온라인 게임하기 → 이 패널 → 빠른 시작(랜덤 매칭) 또는 코드 방 만들기/참가 → 게임.

## 구성요소 (위치 포함)
- **배경**: 우상단 "SIGN IN WITH GOOGLE", 원형 "설정" 버튼, 하단에 가려진 "오프라인 게임하기" 버튼
- **중앙 대형 패널 (연회색)**:
  - 제목: "온라인 게임하기" (패널 상단 중앙)
  - 흰색 가로 버튼: "빠른 시작" (패널 중앙 상부)
  - **하위 섹션 박스 (진회색)**: 제목 "게임 만들기 / 참가하기"
    - 1행: 흰색 버튼 "코드 생성하기" + 그 우측 빈 밑줄(코드 표시 자리, 아직 빈 상태) + "복사" 버튼 + 톱니바퀴 아이콘
    - 2행: 라벨 "코드 입력하기" + 흰색 텍스트 입력창 + "확인" 버튼

## 기능 목록
1. **빠른 시작 (랜덤 매칭)**
   - (a) 대기 중인 아무 상대와 자동 매칭해 게임 시작.
   - (b) 트리거: "빠른 시작" 클릭.
   - (c) mock: 1~2초 가짜 "매칭 중..." 후 봇 상대와 게임 화면으로 이동.
2. **코드 생성하기 (방 만들기)**
   - (a) 초대용 방 코드를 생성해 밑줄 자리에 표시. 방장이 되어 상대 입장 대기.
   - (b) 트리거: "코드 생성하기" 클릭.
   - (c) mock: 랜덤 숫자 문자열 생성해 표시(16:1354 상태로 전환). 상대 입장은 타이머로 가짜 처리 가능.
3. **코드 복사**
   - (a) 생성된 코드를 클립보드에 복사.
   - (b) 트리거: "복사" 클릭.
   - (c) mock: `navigator.clipboard.writeText` + 복사됨 토스트. 코드 미생성 상태에서는 비활성화가 자연스러움.
4. **코드 입력해 참가하기**
   - (a) 상대가 만든 방 코드를 입력해 그 방에 참가.
   - (b) 트리거: 입력창에 코드 입력 후 "확인" 클릭.
   - (c) mock: 코드 검증(형식만) → 매칭 성공 처리 후 게임/대기 화면 이동. 잘못된 코드는 에러 표시.
5. **방 설정 (톱니 아이콘)**
   - (a) 코드 생성 행 옆 톱니 — 만들 방의 규칙(라운드 수/시간, 16:1293 설정 모달) 설정으로 추정.
   - (b) 트리거: 톱니 클릭.
   - (c) mock: 설정 모달 재사용.
6. **패널 닫기** (명시적 X 없음 — 배경 클릭/뒤로가기로 메인 복귀 처리 필요)

## 특이사항
- 같은 이름 프레임 3개 존재: 16:1321(이 프레임, 코드 생성 전), 16:1354(코드 생성 후 — 코드 "34823501249" 표시), 16:1387(다른 담당자 분석, 캔버스상 별개 위치).
- 16:1354와의 유일한 차이는 코드 표시 자리(빈 밑줄 ↔ 실제 코드)이므로 **같은 화면의 상태 변화**임. 별도 페이지가 아니라 한 컴포넌트의 두 상태로 구현.
- 배경 우상단에 여전히 "SIGN IN WITH GOOGLE"이 보이지만, 이 패널은 로그인 후에만 접근 가능하므로 와이어프레임 배경 재사용에 따른 잔재로 추정 (로그인 상태라면 프로필 표시가 맞음 — 16:1474 참고).
