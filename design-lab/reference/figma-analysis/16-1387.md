# 16:1387 — 온라인 게임하기 (매칭 접속 중)

## 역할 추정
온라인 매칭 플로우의 첫 단계. 메인 화면에서 "온라인 게임하기"를 누른 직후, 서버/매칭 시스템에 접속하는 로딩 상태를 보여주는 화면(또는 메인 위에 뜬 상태 패널). 다음 단계는 16:1403(플레이어 대기 중).

## 구성요소
- 우측 상단: "SIGN IN WITH GOOGLE" 버튼 (Google G 로고 포함) — 로그인 전 공통 헤더로 보임
- 우측 최상단 모서리: 원형 "설정" 버튼
- 화면 중앙: 큰 회색 패널(모달/카드)
  - 패널 상단 중앙 제목: "온라인 게임하기"
  - 패널 중앙 텍스트: "게임에 접속 중입니다"
- 패널 하단(패널과 겹쳐서 아래로 삐져나옴): "오프라인 게임하기" 버튼 — 메인 화면의 버튼이 모달 뒤에 남아있는 레이아웃으로 추정

## 기능 목록
1. 온라인 매칭 접속
   - (a) 매칭 서버에 접속을 시도하고 진행 상태를 표시
   - (b) 트리거: 메인에서 "온라인 게임하기" 클릭
   - (c) mock: 버튼 클릭 시 이 상태 패널 표시 → setTimeout(1~2초) 후 "플레이어 대기 중" 상태(16:1403)로 전환
2. Google 로그인 (헤더)
   - (a) Google 계정 로그인 시작
   - (b) 트리거: SIGN IN WITH GOOGLE 클릭
   - (c) mock: 클릭 시 가짜 로그인 처리(닉네임 작성 화면으로 이동)
3. 설정 열기
   - (a) 설정 패널/모달 열기
   - (b) 트리거: 우측 상단 "설정" 원형 버튼 클릭
   - (c) mock: 빈 설정 모달 토글

## 특이사항
- 같은 이름의 프레임 16:1403과 쌍을 이룸: 1387은 "게임에 접속 중입니다"(접속 단계, 취소 버튼 없음), 1403은 "플레이어 대기 중"(매칭 대기 단계, "취소하기" 버튼 있음). 즉 매칭 플로우의 상태 변형 2종.
- 로그인 전 상태인데(SIGN IN WITH GOOGLE 노출) 온라인 게임 화면이 그려져 있음 — 헤더가 공통 컴포넌트로 복붙된 와이어프레임 관행일 가능성. 실제 구현에서는 로그인 후에는 프로필/닉네임 표시로 대체 필요.
- "오프라인 게임하기" 버튼이 패널 아래에 반쯤 가려져 있음 → 이 화면은 별도 페이지가 아니라 메인 위에 겹쳐지는 오버레이/모달로 해석하는 것이 자연스러움.
