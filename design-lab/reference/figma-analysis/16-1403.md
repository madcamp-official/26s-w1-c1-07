# 16:1403 — 온라인 게임하기 (플레이어 대기 중)

## 역할 추정
온라인 매칭 플로우의 두 번째 단계. 서버 접속 완료 후 상대 플레이어가 매칭되기를 기다리는 대기 화면. 16:1387(게임에 접속 중입니다)의 다음 상태이며, 매칭 성사 시 게임 화면으로 넘어갈 것으로 추정.

## 구성요소
- 우측 상단: "SIGN IN WITH GOOGLE" 버튼 (Google G 로고 포함)
- 우측 최상단 모서리: 원형 "설정" 버튼
- 화면 중앙: 큰 회색 패널(모달/카드)
  - 패널 상단 중앙 제목: "온라인 게임하기"
  - 패널 중앙(약간 아래) 텍스트: "플레이어 대기 중"
  - 패널 하단 중앙: 진회색 사각 버튼 "취소하기"
- 패널 아래로 반쯤 가려진 "오프라인 게임하기" 버튼 (메인 화면 요소)

## 기능 목록
1. 매칭 대기 상태 표시
   - (a) 상대 플레이어를 찾는 동안 대기 상태를 표시
   - (b) 트리거: 접속 완료(16:1387 다음 자동 전환)
   - (c) mock: 접속 mock 완료 후 이 상태로 전환, 다시 setTimeout(2~4초) 후 "상대 발견 → 게임 시작" 전환. 대기 애니메이션(점 3개 등) 권장
2. 매칭 취소
   - (a) 매칭 대기를 중단하고 메인으로 복귀
   - (b) 트리거: "취소하기" 버튼 클릭
   - (c) mock: 대기 타이머 clear 후 모달 닫고 메인 상태로 복귀
3. Google 로그인 / 설정 (헤더 공통 — 16-1387.md 참조)

## 특이사항
- 16:1387의 상태 변형: 텍스트가 "게임에 접속 중입니다" → "플레이어 대기 중"으로 바뀌고 "취소하기" 버튼이 추가됨. 두 프레임은 같은 컴포넌트의 상태 2종이므로 구현 시 하나의 매칭 모달에 state(connecting | waiting)로 처리하면 됨.
- 취소 버튼이 접속 단계(1387)에는 없음 — 접속 중에는 취소 불가로 그렸는지 단순 누락인지 불명확. mock에서는 두 단계 모두 취소 가능하게 해도 무방하나 원안 충실도 기준으론 대기 단계에만 노출.
- 여전히 SIGN IN WITH GOOGLE이 보임(로그인 전 헤더 복붙 추정) — 온라인 매칭은 사실상 로그인 후 기능일 가능성이 높으므로 구현 시 확인 필요.
