# 16:1419 — 메인 - 로그인 되기 이전 (Google OAuth 로그인 화면)

## 역할 추정
프레임 이름은 "메인 - 로그인 되기 이전"이지만, 실제 내용은 MADPUMP 자체 화면이 아니라 **Google 계정 로그인 페이지(OAuth 리다이렉트 화면)의 목업**이다. 유저 플로우상 메인에서 "SIGN IN WITH GOOGLE" 클릭 → 이 Google 로그인 페이지로 이동 → 완료 후 닉네임 작성(16:1422)으로 이어지는 중간 단계를 표현한 것.

## 구성요소 (Google 표준 로그인 카드 재현)
- 중앙 카드(흰 배경, 테두리)
  - Google 로고
  - 제목: "로그인"
  - 부제: "Gmail로 이동"
  - 입력창: 라벨 "이메일 또는 휴대전화" (파란 테두리 포커스 상태)
  - 링크: "이메일을 잊으셨나요?"
  - 안내문: "내 컴퓨터가 아닌가요? 게스트 모드를 사용하여 비공개로 로그인하세요." + 링크 "게스트 모드 사용 방법 자세히 알아보기"
  - 좌하단 링크: "계정 만들기" / 우하단 파란 버튼: "다음"
- 카드 하단 바깥: 언어 선택 드롭다운 "한국어 ▾", 링크 "도움말", "개인정보처리방침", "약관"

## 기능 목록
1. Google OAuth 인증
   - (a) Google 계정으로 인증하여 앱에 로그인
   - (b) 트리거: 메인 헤더의 "SIGN IN WITH GOOGLE" 클릭으로 이 페이지 도달
   - (c) mock: 실제 Google 페이지를 만들 필요 없음. "SIGN IN WITH GOOGLE" 클릭 시 가짜 지연(0.5초) 후 로그인 성공 처리하고 닉네임 작성 화면으로 이동. (원하면 간단한 가짜 Google 스타일 모달로 흉내 가능하나 필수 아님)

## 특이사항
- 이 프레임은 우리가 디자인/구현할 화면이 아님 — 외부 서비스(Google) 화면을 플로우 설명용으로 붙여넣은 것. 최종 제품에서는 OAuth 리다이렉트 또는 popup으로 대체됨.
- 이름("메인 - 로그인 되기 이전")과 내용(Google 로그인 페이지)이 불일치. 실제 "로그인 전 메인" 화면은 다른 프레임(온라인/오프라인 게임하기 버튼 + SIGN IN WITH GOOGLE 헤더가 있는 화면)일 가능성이 큼. 플로우 해석 시 주의.
- 시사점: 로그인 수단은 Google 계정 단일. 자체 이메일/비밀번호 가입 없음.
