# 16:1422 — 회원가입 - 닉네임 작성하기 (기본/빈 상태)

## 역할 추정
Google 로그인 직후 최초 1회 진행하는 프로필(닉네임) 등록 화면. 몰입캠프 분반용이므로 이름과 분반을 받는다. 이 프레임은 아무것도 입력하지 않은 기본 상태. 완료 시 로그인된 메인으로 이동할 것으로 추정.

## 구성요소
- 우측 상단: "SIGN IN WITH GOOGLE" 버튼 (헤더 공통)
- 우측 최상단 모서리: 원형 "설정" 버튼
- 중앙 큰 회색 패널(카드)
  - 상단 중앙 제목: "What's your name?"
  - 1행: 좌측 라벨 "이름 :" + 우측 흰색 입력창(비어 있음)
  - 2행: 좌측 라벨 "분반 :" + 우측 흰색 입력창(비어 있음)
  - 하단 중앙: 테두리만 있는 사각 버튼 "확인"

## 기능 목록
1. 이름(닉네임) 입력
   - (a) 게임에서 표시될 이름 입력
   - (b) 트리거: 입력창 포커스 후 타이핑
   - (c) mock: controlled input, 상태에 저장
2. 분반 입력
   - (a) 몰입캠프 분반 정보 입력 (예: 1분반)
   - (b) 트리거: 입력창 타이핑 (또는 select로 개선 가능)
   - (c) mock: controlled input. 와이어프레임은 자유 텍스트이지만 분반은 옵션이 유한하므로 드롭다운화 검토
3. 확인(제출)
   - (a) 이름 중복 검사 후 프로필 저장, 성공 시 메인으로 이동
   - (b) 트리거: "확인" 버튼 클릭
   - (c) mock: 미리 정한 금지 이름 목록(예: "test")이면 중복 에러 상태(16:1923)로, 아니면 저장 후 메인 이동. 빈 입력 시 제출 방지
4. 헤더 Google 로그인 / 설정 (공통 — 16-1387.md 참조)

## 특이사항
- 같은 이름 프레임 3종의 상태 변형 세트 중 "빈 상태":
  - 16:1422 = 빈 입력 (본 프레임)
  - 16:2475 = 이름에 "test" 입력된 상태
  - 16:1923 = 이름 중복 에러 표시 상태
- 회원가입 화면인데 헤더에 SIGN IN WITH GOOGLE이 남아 있음(이미 로그인된 시점이므로 모순) — 공통 헤더 복붙으로 추정, 구현 시 로그인 상태 반영 필요.
- 제목이 영어("What's your name?")이고 라벨은 한국어 — 최종 카피 통일 필요.
