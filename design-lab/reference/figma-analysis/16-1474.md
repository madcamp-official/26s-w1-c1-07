# 16:1474 — 메인 - 로그인 한 뒤

PNG: design-lab/reference/figma/16-1474__메인_-_로그인_한_뒤.png

## 역할 추정
로그인 완료 후 진입하는 메인 홈(로비) 화면. 유저 플로우상 [로그인] → **[메인]** → [온라인/오프라인 게임 진입] 의 허브.

## 구성요소 (위치와 함께)
- 우상단 헤더 라인:
  - 텍스트 "OOO님 안녕하세요" (로그인한 유저 닉네임 인사말)
  - 텍스트 버튼 "로그아웃"
  - 원형 버튼 "설정" (가장 우측, 원형 아이콘 자리)
- 중앙(약간 좌측): 대형 로고 텍스트 "MADPUMP"
- 우측: 회색 패널 박스, 상단 라벨 "<1분반 내 리더보드>" — 분반(1분반) 내 랭킹 리스트 자리
- 중앙 하단: 버튼 2개 세로 배치
  - "온라인 게임하기"
  - "오프라인 게임하기"
- "오프라인 게임하기" 버튼 우측 하단에 톱니바퀴(설정) 아이콘 하나 더 존재 (버튼별 옵션 or 장식 주석일 수 있음)

## 기능 목록
1. 유저 인사말 표시
   - (a) 로그인한 유저의 닉네임 표시
   - (b) 페이지 로드 시 세션/유저 정보 조회
   - (c) mock: localStorage 또는 mock user 객체에서 nickname 렌더
2. 로그아웃
   - (a) 세션 종료 후 로그인 전 메인으로 이동
   - (b) "로그아웃" 클릭
   - (c) mock: mock 세션 클리어 + 라우팅
3. 설정
   - (a) 설정 화면/모달 진입 (프로필, 사운드 등 추정)
   - (b) 우상단 원형 "설정" 클릭
   - (c) mock: 설정 모달 오픈 (내용은 placeholder 가능)
4. 분반 리더보드 표시
   - (a) 1분반(내 분반) 내 유저 랭킹 리스트 표시
   - (b) 페이지 로드 시 자동 조회
   - (c) mock: mock 랭킹 배열(닉네임+점수/레이팅) 렌더. 데이터 없으면 빈 상태 표시(가짜 데이터 생성 금지 원칙 유의 — mock 단계에서는 명시적 mock 데이터로)
5. 온라인 게임 진입
   - (a) 온라인 매칭(대기/매치메이킹) 플로우 시작
   - (b) "온라인 게임하기" 클릭
   - (c) mock: 매칭 대기 화면으로 라우팅, setTimeout 으로 가짜 매칭 성사
6. 오프라인 게임 진입
   - (a) 로컬(같은 기기 2인 또는 vs AI 추정) 게임 시작 플로우
   - (b) "오프라인 게임하기" 클릭
   - (c) mock: 게임 선택/게임 화면으로 바로 라우팅

## 특이사항
- "<1분반 내 리더보드>" 는 꺾쇠 표기로 보아 placeholder 주석 성격 — 실제 리스트 UI는 미정.
- 하단 톱니 아이콘의 용도가 불명확 (우상단 설정과 중복). 게임 옵션 설정일 가능성.
- 로그인 전 메인 프레임이 별도로 존재할 것으로 추정(이 프레임 이름이 "로그인 한 뒤"이므로 대응 변형 프레임 존재).
