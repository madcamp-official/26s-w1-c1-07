# 16:1489 — 게임1

PNG: design-lab/reference/figma/16-1489__게임1.png

## 역할 추정
게임1(숫자 맞추기/타겟 숫자 도달형 대결 게임)의 인게임 플레이 화면. 메인 → 매칭 → **게임1 플레이** 단계. 두 플레이어가 각자 현재 숫자를 조작해 중앙의 타겟 숫자에 도달(또는 근접)하는 대결로 추정.

## 구성요소 (위치와 함께)
- 좌상단: 회색 박스 "프로필" (P1 플레이어 프로필 영역)
- 우상단: 회색 박스 "프로필" (P2 플레이어 프로필 영역)
- 상단 중앙: 가로 바 "(count down)" — 라운드 제한시간 카운트다운 표시 자리
- 중앙: 큰 회색 박스 "타겟숫자" — 이번 라운드의 목표 숫자 표시
- 좌하단: 회색 박스 "P1 현재숫자"
- 우하단: 회색 박스 "P2 현재숫자", 그 위에 굵은 주석 텍스트 "this is you" (이 화면의 시점 유저 = P2)
- 하단 중앙: 원형 패드(펌프 버튼) 2개
  - 좌측 패드: 아래 방향 화살표 + 라벨 "P" (감소 키)
  - 우측 패드: 위 방향 화살표 + 라벨 "Q" (증가 키)

## 기능 목록
1. 카운트다운 타이머
   - (a) 라운드 남은 시간 표시, 0이 되면 라운드 종료/판정
   - (b) 게임 시작 시 자동 시작
   - (c) mock: setInterval 1초 감소, 0에서 결과 화면 전환
2. 타겟 숫자 제시
   - (a) 두 플레이어가 도달해야 할 목표 숫자 표시
   - (b) 라운드 시작 시 서버(또는 시드)에서 생성
   - (c) mock: 난수 생성해 표시 (양쪽 동일 시드)
3. 내 숫자 조작 (P/Q 입력)
   - (a) Q(위 화살표)=현재숫자 증가, P(아래 화살표)=현재숫자 감소
   - (b) 키보드 P/Q 키 입력 또는 화면 패드 터치/클릭
   - (c) mock: keydown 리스너 + 버튼 onClick, 내 현재숫자 state 갱신
4. 내 현재숫자 표시 (P2, "this is you")
   - (a) 내 조작 결과를 실시간 표시
   - (b) 입력 즉시 갱신
   - (c) mock: 로컬 state 바인딩
5. 상대 현재숫자 표시 (P1)
   - (a) 상대의 진행 상황 실시간 표시
   - (b) 네트워크(소켓) 수신
   - (c) mock: 랜덤 워크로 타겟에 접근하는 가짜 상대 시뮬레이션 (mock임을 코드상 명시)
6. 플레이어 프로필 표시
   - (a) 양측 닉네임/아바타(및 승수 등) 표시
   - (b) 매칭 완료 시 데이터 수신
   - (c) mock: mock 플레이어 객체 렌더
7. 승패 판정 (암시)
   - (a) 타겟 도달 선착 또는 시간 종료 시 근접자 승리 판정
   - (b) 도달 이벤트 or 카운트다운 0
   - (c) mock: 로컬 판정 로직 후 결과 화면 이동

## 특이사항
- "this is you" 주석: 이 프레임은 P2 시점 기준의 레이아웃 설명. 즉 내 정보는 우측/우하단에 배치된다는 규칙.
- 17:1178 프레임("게임1")과 픽셀 단위로 동일한 내용 (파일 크기까지 동일). 페이지/버전 복제본으로 보임.
- P=감소(↓), Q=증가(↑) 키 매핑이 화살표로 명시됨. 펌프 패드 모양은 리듬게임(PUMP) 발판 메타포.
