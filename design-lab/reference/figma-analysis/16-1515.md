# 16:1515 — 게임2 - P2

PNG: design-lab/reference/figma/16-1515__게임2_-_P2.png

## 역할 추정
게임2(노트 낙하형 리듬/반응 대결 게임)의 **P2 시점** 인게임 화면. P1이 위에서 노트를 떨어뜨리고(공격), P2(나)는 아래에서 그 노트를 P/Q 패드로 받아내는 구도로 추정. 16:1543("게임2 - P1")의 반대 시점 화면.

## 구성요소 (위치와 함께)
- 상단: 화면 폭 전체 가로선 + 그 중앙에 원형 배지 "P1" (상대 위치 라인. P1 배지가 좌우로 움직이며 노트 발사 위치를 나타내는 것으로 추정)
- 중앙 상~중부: 낙하 중인 노트 3개 (사각형 위에 빨간 반원이 달린 캡슐 모양) — P1 위치에서 아래로 대각선 궤적으로 떨어지는 중
- 중앙: 반투명(비활성 표시) 원형 패드 2개
  - 좌측 패드: 좌향 화살표 + 라벨 "P"
  - 우측 패드: 우향 화살표 + 라벨 "Q"
  - ※ 게임1과 달리 화살표가 좌/우 방향. 흐리게 처리된 것은 "아직 입력 불가" 또는 "이건 조작 UI 위치 표시" 주석일 수 있음
- 하단: 화면 폭 전체 가로선 + 우측에 원형 배지 "P2" (내 위치 라인. 카운트다운 표시 없음)

## 기능 목록
1. 상대(P1) 위치 표시
   - (a) 상단 라인 위에서 P1 배지의 좌우 위치를 실시간 표시 (노트 발사 지점)
   - (b) 상대 입력 수신
   - (c) mock: 가짜 P1 이동 시뮬레이션 (사인파/랜덤)
2. 노트 낙하 렌더링
   - (a) P1이 발사한 노트가 위에서 아래(P2 라인)로 떨어지는 애니메이션
   - (b) 상대 발사 이벤트 수신 시 스폰
   - (c) mock: requestAnimationFrame 물리 루프, mock 발사 이벤트 큐
3. 내(P2) 좌우 이동
   - (a) P(←)/Q(→) 키로 하단 라인 위 내 배지를 좌우로 이동
   - (b) 키보드 P/Q 또는 패드 터치
   - (c) mock: keydown 리스너로 x좌표 갱신
4. 노트 판정 (받기/맞기)
   - (a) 노트가 하단 라인 도달 시 내 위치와의 충돌 판정 → 회피/피격(또는 캐치) 처리, 점수·체력 반영
   - (b) 노트 y가 라인 도달하는 프레임
   - (c) mock: AABB/거리 기반 충돌 체크 + 점수 state
5. 내 패드 입력 UI
   - (a) 모바일/터치용 P·Q 온스크린 버튼
   - (b) 터치/클릭
   - (c) mock: 버튼 press 시 키 입력과 동일 핸들러 호출

- 16:1543("게임2 - P1")과의 차이: P1 화면에는 상단에 "(count down)" 바가 있고 P/Q 패드 화살표 위에 노트가 겹쳐 그려져 "패드로 노트를 쏘는" 뉘앙스인 반면, 이 P2 화면은 카운트다운이 없고 노트가 P1 라인에서 이미 떨어져 내려오는 중. 즉 P1=공격(발사) 턴, P2=수비(회피/받기) 턴의 비대칭 시점.
- 패드 화살표가 게임1(↑↓)과 달리 ←→ — 게임2에서는 P/Q가 좌우 이동 키.
- 카운트다운 부재는 실수 누락일 수도 있으므로 구현 시 공용 타이머 포함 여부를 팀 확인 필요.
