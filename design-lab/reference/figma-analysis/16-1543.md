# 16:1543 — 게임2 - P1

PNG: design-lab/reference/figma/16-1543__게임2_-_P1.png

## 역할 추정
게임2의 **P1(공격/발사) 시점** 인게임 화면. P1(나)이 P/Q 패드를 눌러 노트를 아래(P2 라인 방향)로 발사하고, P2는 하단 라인에서 이를 받는/피하는 구도. 16:1515·17:1206("게임2 - P2")의 반대 시점.

## 구성요소 (위치와 함께)
- 최상단 중앙: 가로 바 "(count down)" — 라운드 제한시간 표시 자리 (P2 프레임에는 없음)
- 상단: 화면 폭 전체 가로선 + 중앙 원형 배지 "P1" (내 위치 라인)
- 중앙: 흐리게 처리된 원형 패드 2개
  - 좌측 패드: 좌향 화살표 + 라벨 "P"
  - 우측 패드: 우향 화살표 + 라벨 "Q"
- 노트(빨간 반원 캡슐) 4개:
  - P1 배지 아래로 3개가 세로로 이어지며 P 패드 위를 관통해 낙하 중 (내가 발사한 노트들의 궤적)
  - Q 패드 위에 흐린(반투명 연분홍) 노트 1개 — 미발사/장전 상태 또는 발사 예정 위치 프리뷰로 추정
- 하단: 화면 폭 전체 가로선 + 우측 원형 배지 "P2" (상대 위치)

## 기능 목록
1. 카운트다운 타이머
   - (a) 라운드 남은 시간 표시, 종료 시 턴 교대 또는 결과 판정
   - (b) 라운드 시작 시 자동
   - (c) mock: setInterval, 0에서 시점 전환(P1↔P2) 또는 결과 화면
2. 내(P1) 좌우 이동
   - (a) 상단 라인 위에서 내 배지를 좌우 이동해 발사 위치 조준
   - (b) 키보드 P(←)/Q(→) 또는 패드 (이동 vs 발사 역할 배분은 와이어프레임상 모호 — 확인 필요)
   - (c) mock: keydown으로 x좌표 갱신
3. 노트 발사
   - (a) 현재 내 위치에서 노트를 아래로 발사 (P2를 맞추는 공격)
   - (b) 패드/키 입력 (P·Q가 각각 다른 레인/종류 발사일 가능성)
   - (c) mock: 입력 시 노트 객체 스폰, rAF 루프로 낙하
4. 발사된 노트 궤적 표시
   - (a) 내가 쏜 노트들이 P2 라인까지 떨어지는 애니메이션
   - (b) 발사 직후 자동
   - (c) mock: 노트 배열 상태 + 프레임 루프
5. 장전/쿨다운 표시 (추정)
   - (a) Q 패드 위 흐린 노트 = 다음 발사 가능 노트 or 쿨다운 상태 시각화
   - (b) 발사 후 일정 시간 경과
   - (c) mock: 쿨다운 타이머에 따라 opacity 토글
6. 상대(P2) 위치 표시
   - (a) 하단 라인 위 P2 배지의 실시간 위치 표시 (조준 대상)
   - (b) 상대 입력 수신
   - (c) mock: 가짜 P2 이동 시뮬레이션
7. 명중 판정
   - (a) 노트가 하단 라인 도달 시 P2와 충돌하면 득점/피해
   - (b) 노트 도달 프레임
   - (c) mock: 충돌 체크 + 점수 state (판정 자체는 게임 코어 공용 모듈)

## 특이사항
- P2 시점 프레임(16:1515/17:1206)과의 차이: (1) "(count down)" 바가 이 프레임에만 있음, (2) 노트 궤적이 P 패드를 관통하며 그려져 "패드 입력 → 발사" 관계를 표현, (3) Q 패드 위 흐린 노트 존재. 나머지 레이아웃(상하 라인, P1/P2 배지, 패드)은 동일.
- P1=공격, P2=수비의 비대칭 화면 설계. 실제 게임이 턴제(공수 교대)인지 동시 진행(각자 자기 시점)인지 와이어프레임만으로는 미확정 — 기능정의서와 대조 필요.
