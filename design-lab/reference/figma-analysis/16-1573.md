# 16:1573 — "게임2" (게임2 펜싱 대결 — P 연타/변경 도식)

이미지: `design-lab/reference/figma/16-1573__게임2.png`

## 역할 추정
게임2의 인게임 화면을 **측면 뷰 펜싱 대결** 컨셉으로 그린 와이어프레임. 좌측 캐릭터는 검(공격), 우측 캐릭터는 방패(방어)를 든 상태. 상단에는 조작 요소 도식(변경 버튼 + P 패드 연타)이 배치. 유저 플로우상 게임2 플레이 단계의 핵심 화면.

## 구성요소 (위치와 텍스트)
- 상단 중앙(좌): 원형 버튼, 텍스트 **"변경"** — 상태(무기/스탠스) 전환 버튼으로 추정
- 상단 중앙(우): 원반 패드, 라벨 **"P"**, 그 위로 **아래 방향 화살표가 여러 겹 쌓여 내려오는 표현** — P 키를 연타해 패드를 두드리는 동작(연타 게이지) 도식
- 중앙~하단: 측면 뷰 스틱 캐릭터 2명
  - 좌측 캐릭터: **검(레이피어)을 앞으로 뻗은 자세** (공격 스탠스)
  - 우측 캐릭터: **부채꼴 방패를 든 자세** (방어 스탠스)
- 하단: 두 캐릭터가 올라선 큰 회색 플랫폼(무대). 플랫폼 안 하단 중앙에 텍스트 **"(count down)"** — 남은 시간 표시가 무대 영역에 들어감

## 기능 목록
1. **P 키 연타 (공격/전진 게이지)**
   - (a) P 키를 반복 입력해 공격 게이지를 올리거나 캐릭터를 전진/공격시킴
   - (b) 트리거: keydown `P` 연타 (겹쳐 내려오는 화살표 = 연속 입력 표현)
   - (c) mock: keydown 카운트 → 연타 속도에 비례해 게이지/전진량 증가, 애니메이션 반영
2. **"변경" 버튼 — 스탠스(공격↔방어) 전환**
   - (a) 내 캐릭터의 모드를 전환 (검=공격 ↔ 방패=방어로 추정)
   - (b) 트리거: 해당 키 입력(아마 Q, 16:1607 프레임 참조) 또는 버튼 클릭
   - (c) mock: 토글 상태 변수 + 캐릭터 스프라이트 교체
3. **캐릭터 스탠스 시각화**
   - (a) 각 플레이어의 현재 모드(검/방패)를 캐릭터 포즈로 표시
   - (b) 트리거: 스탠스 상태 변경 시
   - (c) mock: 상태별 SVG/이미지 스왑
4. **카운트다운 타이머**
   - (a) 라운드 남은 시간 표시 (무대 하단)
   - (b) 트리거: 라운드 시작 시 자동
   - (c) mock: setInterval, 0이면 판정/결과 전환
5. **대결 판정 (공격 vs 방어 상호작용)**
   - (a) 공격 측 연타가 방어를 뚫으면 득점/승리, 방어 성공 시 무효 — 가위바위보식 스탠스 상성으로 추정
   - (b) 트리거: 연타 게이지 임계 도달 또는 캐릭터 충돌
   - (c) mock: 게이지 차이/스탠스 조합으로 승패 계산, 봇 상대 시뮬레이션

## 특이사항
- **17:1266과 픽셀 단위로 동일한 프레임** (내보내기 크기 23,099 vs 23,101 bytes, 내용 동일). 한쪽은 복제본.
- 상단의 "변경"+P 패드 도식은 실제 UI 배치라기보다 조작 방법 주석(legend)에 가까움 — 실제 화면에서는 캐릭터/무대가 본체.
- 16:1607(17:1302)에서는 Q 패드 2개에 검/방패 아이콘이 있음 → "변경"이 Q 키에 대응하고, P는 연타(행동), Q는 스탠스 변경이라는 조작 체계로 종합 추정됨.
- 17:1234(상하 분할 노트 낙하)와 같은 이름의 게임2지만 구도가 전혀 다름 — 컨셉 버전 차이 가능성, 확정안 확인 필요.
