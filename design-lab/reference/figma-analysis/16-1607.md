# 16:1607 — "게임2" (게임2 펜싱 대결 — Q 키 스탠스 표시 도식)

이미지: `design-lab/reference/figma/16-1607__게임2.png`

## 역할 추정
게임2 펜싱 대결 화면의 **다른 상태/도식** — 양 플레이어 각각의 Q 패드와 현재 장비(검/방패)를 보여주는 프레임. 16:1573("변경"+P 연타 도식)과 짝을 이루어 Q 키의 역할(스탠스 전환/표시)을 설명하는 것으로 추정. 유저 플로우상 동일한 게임2 플레이 화면의 변형 상태.

## 구성요소 (위치와 텍스트)
- 상단 좌: 원반 패드, 라벨 **"Q"**, 패드 위에 **검(레이피어) 아이콘이 꽂힌 형태** — 좌측 플레이어의 현재 스탠스 = 공격(검)
- 상단 우: 원반 패드, 라벨 **"Q"**, 패드 위에 **부채꼴 방패 아이콘** — 우측 플레이어의 현재 스탠스 = 방어(방패)
- 중앙~하단: 측면 뷰 스틱 캐릭터 2명 (16:1573과 동일 포즈)
  - 좌측: 검을 앞으로 뻗은 캐릭터
  - 우측: 방패를 든 캐릭터
- 하단: 무대 플랫폼 (이 프레임에는 "(count down)" 텍스트 없음 — 빈 플랫폼)

## 기능 목록
1. **Q 키 — 스탠스(무기) 전환/발동**
   - (a) Q 패드가 각 플레이어의 현재 장비(검/방패)를 표시하며, Q 입력으로 전환하는 것으로 추정 (16:1573의 "변경" 버튼과 대응)
   - (b) 트리거: keydown `Q`
   - (c) mock: Q 입력 시 attack↔defense 토글, 패드 위 아이콘과 캐릭터 포즈 동시 갱신
2. **플레이어별 현재 스탠스 인디케이터**
   - (a) 좌/우 상단 Q 패드 위 아이콘으로 자신·상대의 현재 모드를 한눈에 표시
   - (b) 트리거: 스탠스 상태 변경 시 실시간 갱신
   - (c) mock: 상태 변수 구독 → 아이콘 스왑, 상대 상태는 mock 채널/봇에서 수신
3. **캐릭터 포즈 동기화**
   - (a) 스탠스에 맞는 캐릭터 그래픽(검 찌르기 자세/방패 방어 자세) 표시
   - (b) 트리거: 스탠스 변경
   - (c) mock: 상태별 SVG 교체(+간단 전환 애니메이션)
4. **공격 vs 방어 상성 대결** (16:1573과 공유)
   - (a) 검(공격)과 방패(방어)의 조합으로 데미지/무효 판정
   - (b) 트리거: P 연타 게이지 임계/충돌 시점의 양측 스탠스 비교
   - (c) mock: (내 스탠스, 상대 스탠스) 조합 테이블로 판정

## 특이사항
- **17:1302와 동일한 복제 프레임** (렌더 동일, 파일 크기 20,579 vs 20,722 bytes 수준 차이).
- 16:1573과의 차이: 상단 도식이 "변경 버튼 + P 연타" → "Q 패드 2개 + 검/방패 아이콘"으로 바뀌고, 플랫폼의 "(count down)" 텍스트가 없음. 즉 두 프레임은 같은 화면의 조작 설명을 P(연타/행동)와 Q(스탠스) 관점에서 나눠 그린 것으로 해석됨.
- 좌=검, 우=방패 구도가 캐릭터·아이콘 모두에서 일치 — 스탠스가 비대칭인 순간의 스냅샷.
