# 16:1716 — "관리자 메인페이지" (관리자 대시보드 셸 / 빈 상태)

## 역할 추정
**관리자(운영자)용 메인 레이아웃**. 좌측 사이드바 내비게이션 + 우측 빈 콘텐츠 영역으로 구성된 관리자 대시보드의 기본 셸. 관리자 로그인(16:2320) 성공 직후 도착하는 첫 화면. 콘텐츠 영역이 비어 있어 메뉴 선택 전의 초기 상태를 표현.

## 구성요소 (위치와 함께)
- **좌측 사이드바** (화면 좌측 약 21% 폭, 회색 배경, 전체 높이):
  - 상단: 로고 텍스트 `MADPUMP` (굵은 글씨)
  - 메뉴 항목 1: `그룹 관리`
  - 메뉴 항목 2: `전체 인원 관리`
  - 최하단: `로그아웃`
- **우측 콘텐츠 영역**: 완전히 비어 있음(흰색)

## 기능 목록
1. **사이드바 내비게이션 — 그룹 관리**
   - (a) 그룹(분반) 목록/검색 화면으로 이동 (16:1724 "그룹 관리 - 그룹 찾기"가 목적지)
   - (b) 트리거: `그룹 관리` 클릭
   - (c) mock: 라우터 링크(`/admin/groups`), 활성 메뉴 하이라이트
2. **사이드바 내비게이션 — 전체 인원 관리**
   - (a) 전체 사용자 목록/관리 화면으로 이동
   - (b) 트리거: `전체 인원 관리` 클릭
   - (c) mock: `/admin/users` 라우트 + mock 사용자 테이블
3. **로그아웃**
   - (a) 관리자 세션 종료 후 관리자 로그인 화면(16:2320)으로 복귀
   - (b) 트리거: 사이드바 하단 `로그아웃` 클릭
   - (c) mock: 로컬 세션 상태 초기화 + `/admin/login` 리다이렉트
4. **로고 → 홈**
   - (a) `MADPUMP` 로고 클릭 시 관리자 메인으로 복귀(관례상)
   - (b) 트리거: 로고 클릭
   - (c) mock: `/admin` 라우트 링크

## 특이사항
- 같은 이름 "관리자 메인페이지" 프레임이 16:2320에도 있으나, **16:2320은 관리자 로그인 폼(ID/PW)**이고 이 프레임은 로그인 후 대시보드 셸 — 이름은 같지만 역할이 다름(플로우의 전/후 단계).
- 16:1724("그룹 관리 - 그룹 찾기")는 이 프레임과 동일한 사이드바를 공유하고 콘텐츠 영역에 그룹 테이블이 채워진 변형 — 사이드바는 공용 레이아웃 컴포넌트로 구현하면 됨.
