## 구성요소 (위치와 함께)
- **좌측 사이드바** (16:1716과 동일한 공용 레이아웃):
  - 상단 로고 `MADPUMP`
  - 메뉴 `그룹 관리` (현재 화면), `전체 인원 관리`
  - 최하단 `로그아웃`
- **콘텐츠 영역 상단**: 검색 바
  - 라벨 `그룹명 검색` + 텍스트 입력창 + 돋보기(검색) 아이콘 버튼
- **콘텐츠 영역 중앙**: 그룹 목록 테이블 (5개 열 + 다수의 빈 행)
  - 헤더: `그룹명` | `인원` | `생성일시` | `공개 여부` | `점수 및 랭킹`
  - 행1 (예시 데이터): `몰입캠프 1분반` | `15` | `2026-07-02 19:00` | `공개` | `10023(1순위)`
  - 행2: `몰입캠프 2분반` (다른 열 비어 있음)
  - 행3: `몰입캠프 3분반` (다른 열 비어 있음)
  - 이하 빈 행 약 8~9개 (테이블 골격)
  - 테이블이 우측 화면 끝까지 이어짐(5번째 열 이후 잘린 듯한 여백 있음)

## 기능 목록
1. **그룹명 검색**
   - (a) 그룹명 키워드로 테이블 필터링
   - (b) 트리거: 입력 후 돋보기 아이콘 클릭 또는 Enter
   - (c) mock: mock 그룹 배열에 대해 `includes` 필터, 즉시(onChange) 필터링도 무방
2. **그룹 목록 조회 (테이블)**
   - (a) 전체 그룹의 그룹명/인원/생성일시/공개 여부/점수 및 랭킹을 표 형태로 표시
   - (b) 트리거: 화면 진입 시 자동 로드
   - (c) mock: 하드코딩 그룹 데이터(몰입캠프 1~3분반 등) 렌더. 데이터 없는 셀은 빈 칸 유지(가짜 값 채우지 않기)
3. **그룹 상세 진입 (추정)**
   - (a) 특정 그룹 행 클릭 시 해당 그룹의 인원/설정 관리 화면으로 이동
   - (b) 트리거: 테이블 행 클릭
   - (c) mock: 행 클릭 → `/admin/groups/:id` 라우트 (다른 담당자의 그룹 상세 프레임과 연결 확인 필요)
4. **공개 여부 표시/관리**
   - (a) 그룹이 공개/비공개인지 표시 (관리 화면이므로 토글 가능성 있음)
   - (b) 트리거: 조회 시 표시; 토글이라면 셀 클릭
   - (c) mock: 텍스트 표시(공개/비공개), 토글은 상세 화면에서 처리
5. **점수 및 랭킹 표시**
   - (a) 그룹 누적 점수와 그룹 간 순위를 표시 (예: `10023(1순위)`)
   - (b) 트리거: 조회 시 표시
   - (c) mock: mock 점수로 정렬해 `점수(N순위)` 포맷 렌더
