## 역할 추정
관리자(그룹장) 화면. "그룹 관리 → 특정 그룹(몰입캠프 1분반) 상세 → 그룹 내 인원 목록" 화면(16:2111으로 추정)에서, 특정 멤버의 "내보내기" 버튼을 누른 직후의 **강퇴 확인 모달** 상태이다. 유저 플로우상 관리 플로우의 마지막 확인 단계(파괴적 동작 확인 다이얼로그).

## 구성요소
- **좌측 사이드바** (회색 배경, 세로 전체):
  - 상단 로고 텍스트 "MADPUMP"
  - 메뉴 "그룹 관리"
  - 메뉴 "전체 인원 관리"
  - 하단 "로그아웃"
- **본문 상단**:
  - 페이지 제목(그룹명) "몰입캠프 1분반" (볼드, 좌상단)
  - 라벨 "닉네임 검색" + 텍스트 입력창 + 돋보기(검색) 아이콘 버튼
- **멤버 테이블** (본문 중앙, 다수의 빈 행 포함):
  - 헤더 열: "닉네임" | "Google ID" | "가입일시" | "플레이 수" | "관리"
  - 행 1: 박준서 | asdferq@google.com | 2026-07-02 19:00 | (플레이 수 값은 모달에 가려짐) | "내보내기" (빨간색 텍스트 버튼)
  - 행 2: 이종혁 | (모달에 가려짐) | "내보내기"
  - 행 3: 라태형 | (모달에 가려짐) | "내보내기"
  - 이후 빈 행 다수 (고정 높이 테이블 골격)
- **확인 모달** (화면 중앙, 테이블 위에 겹침, 흰 배경 + 테두리):
  - 본문 텍스트: "**박준서**(asdferq@google.com) 님을 **몰입캠프 1분반** 그룹에서 내보내시겠습니까?" (닉네임·그룹명 볼드)
  - 버튼 "확인" (어두운 배경, 좌)
  - 버튼 "취소" (밝은 회색 배경, 우)

## 기능 목록
1. **멤버 내보내기(강퇴) 확인**
   - (a) 선택한 멤버를 그룹에서 제거하기 전 최종 확인
   - (b) 트리거: 테이블 행의 "내보내기" 버튼 클릭 → 모달 표시; "확인" 클릭 시 제거 실행, "취소" 클릭 시 모달 닫기
   - (c) mock: confirm 모달 컴포넌트(대상 닉네임/이메일/그룹명 문자열 삽입). 확인 시 로컬 상태 배열에서 해당 멤버 제거 후 테이블 리렌더
2. **닉네임 검색** (배경 화면 기능)
   - (a) 그룹 내 멤버를 닉네임으로 필터링
   - (b) 트리거: 입력 후 돋보기 클릭(또는 Enter)
   - (c) mock: 클라이언트 사이드 `filter(m => m.nickname.includes(q))`
3. **멤버 목록 조회** (배경 화면 기능)
   - (a) 닉네임/Google ID/가입일시/플레이 수 표시
   - (b) 트리거: 그룹 상세 진입 시 자동 로드
   - (c) mock: 하드코딩된 멤버 배열 렌더
4. **사이드바 네비게이션 / 로그아웃**
   - (a) "그룹 관리", "전체 인원 관리" 화면 전환, 로그아웃
   - (b) 트리거: 각 메뉴 클릭
   - (c) mock: 라우팅(또는 탭 상태 전환), 로그아웃은 메인(비로그인) 화면으로 이동

## 특이사항
- 16:2111 "그룹 관리 - 그룹 내 인원"과 동일 화면의 **모달 오버레이 변형**이다. 베이스 레이아웃(사이드바·검색·테이블)은 동일하고 확인 모달만 추가됨.
- "내보내기" 버튼이 빨간 텍스트인 것은 파괴적 동작임을 표시. 모달의 "확인"이 어두운(주요) 버튼, "취소"가 회색(보조) 버튼으로 일반적 관례와 반대 배치(확인이 좌측).
- 관리자용 화면군(사이드바 있는 레이아웃)은 플레이어용 게임 화면군과 레이아웃 체계가 다름.
