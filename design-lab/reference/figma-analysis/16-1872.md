# 16:1872 — 그룹 관리 - 그룹 찾기 - 검색어 입력 1

- 원본 프레임: id=16:1872, 캔버스 위치 (6247,2217)
- 이미지: `design-lab/reference/figma/16-1872__그룹_관리_-_그룹_찾기_-_검색어_입력_1.png`

## 역할 추정
관리자 "그룹 관리" 섹션의 **그룹 찾기(검색) 결과 화면 — 결과가 있는 경우**. 그룹명 검색창에 "1분반"을 입력해 검색한 뒤, 일치하는 그룹 1건("몰입캠프 1분반")이 테이블에 표시된 상태. 플로우: 그룹 관리(그룹 목록/찾기) → 검색어 입력 → 결과 표시 → 결과 행 클릭 시 그룹 상세(그룹 내 인원, 16:2111)로 진입하는 흐름으로 추정.

## 구성요소
- **좌측 사이드바** (회색, 다른 관리 화면과 동일):
  - "MADPUMP" 로고
  - "그룹 관리"
  - "전체 인원 관리"
  - 하단 "로그아웃"
- **본문 상단**:
  - 라벨 "그룹명 검색" + 텍스트 입력창(입력값: "1분반") + 돋보기(검색) 아이콘 버튼
  - 페이지 제목 없음 (16:1780의 그룹 상세와 달리 그룹명 타이틀이 없음 = 목록/검색 화면)
- **그룹 테이블**:
  - 헤더 열: "그룹명" | "인원" | "생성일시" | "공개 여부"
  - 행 1: 몰입캠프 1분반 | 15 | 2026-07-02 19:00 | 공개
  - 이후 빈 행 다수 (고정 높이 테이블 골격)

## 기능 목록
1. **그룹명 검색**
   - (a) 그룹명 부분 일치로 그룹 목록 필터링
   - (b) 트리거: 검색창 입력 후 돋보기 클릭(또는 Enter)
   - (c) mock: 하드코딩 그룹 배열에 대해 `filter(g => g.name.includes(q))`, 검색 전에는 전체 목록 표시로 처리해도 무방
2. **검색 결과 목록 표시**
   - (a) 그룹명/인원 수/생성일시/공개 여부를 행으로 표시
   - (b) 트리거: 검색 실행 시
   - (c) mock: 상태 배열 map 렌더. 공개 여부는 "공개"/"비공개" 문자열
3. **그룹 상세 진입 (암시적)**
   - (a) 결과 행을 클릭하면 해당 그룹의 "그룹 내 인원" 화면으로 이동 (와이어프레임 흐름상 추정 — 명시적 버튼 없음)
   - (b) 트리거: 행 클릭
   - (c) mock: 행에 onClick → 그룹 상세 뷰로 라우팅
4. **사이드바 네비게이션 / 로그아웃**
   - (a)(b)(c) 16-1780과 동일

## 특이사항
- 16:1893("검색어 입력 2 — 해당하는 그룹명 없는 경우")과 **동일 화면의 상태 변형 쌍**: 이 프레임은 결과 1건 존재 상태, 16:1893은 결과 0건("결과 없음") 상태. 레이아웃·검색창·테이블 헤더는 동일.
- 그룹 내 인원 화면(16:1780/16:2111)의 검색 라벨은 "닉네임 검색"이고 이 화면은 "그룹명 검색" — 같은 검색 UI 패턴의 재사용.
- "공개 여부" 열이 있는 것으로 보아 그룹에 공개/비공개 속성이 존재하며, 검색으로 찾을 수 있는 것은 공개 그룹 위주일 가능성(확정 아님, 이 화면은 관리자 화면이므로 전체 노출일 수도 있음).
