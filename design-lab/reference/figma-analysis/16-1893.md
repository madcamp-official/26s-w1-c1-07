# 16:1893 — 그룹 관리 - 그룹 찾기 - 검색어 입력 2 (해당하는 그룹명 없는 경우)

- 이미지: `design-lab/reference/figma/16-1893__그룹_관리_-_그룹_찾기_-_검색어_입력_2_(해당하는_그룹명_없는_경우).png`

## 역할 추정
관리자 "그룹 관리 → 그룹 찾기" 검색 결과 화면의 **빈 결과(empty state) 변형**. 검색창에 "코르티스"를 입력했지만 일치하는 그룹이 없어 테이블 본문 대신 "결과 없음" 문구가 표시된 상태. 16:1872(결과 있음)와 짝을 이루는 상태 분기.

## 구성요소
- **좌측 사이드바** (다른 관리 화면과 동일):
  - "MADPUMP" 로고
  - "그룹 관리"
  - "전체 인원 관리"
  - 하단 "로그아웃"
- **본문 상단**:
  - 라벨 "그룹명 검색" + 텍스트 입력창(입력값: "코르티스") + 돋보기(검색) 아이콘 버튼
- **그룹 테이블 (빈 상태)**:
  - 헤더 행만 표시: "그룹명" | "인원" | "생성일시" | "공개 여부"
  - 본문 행 없음 — 테이블 아래 중앙에 빈 상태 텍스트 "**결과 없음**"

## 기능 목록
1. **그룹명 검색** — 16-1872와 동일
   - (a) 그룹명으로 그룹 필터링
   - (b) 트리거: 입력 후 돋보기 클릭/Enter
   - (c) mock: `filter(...)` 결과가 빈 배열
2. **빈 결과 상태 표시**
   - (a) 검색 결과가 0건일 때 가짜 행 대신 "결과 없음" 문구를 명시적으로 표시 (정직한 empty state)
   - (b) 트리거: 검색 실행 후 결과 배열 length === 0
   - (c) mock: `results.length === 0 ? <결과 없음 문구> : <테이블 행들>` 조건부 렌더. 헤더는 유지
3. **사이드바 네비게이션 / 로그아웃** — 다른 관리 화면과 동일

## 특이사항
- 16:1872와 **동일 화면·동일 레이아웃의 결과 유무 상태 쌍**. 차이는 (1) 검색어("1분반" vs "코르티스"), (2) 테이블 본문: 결과 행 + 빈 행 골격 vs 헤더만 + "결과 없음" 텍스트.
- 결과 있음 화면(16:1872)은 결과 아래에 빈 행 골격을 그리지만, 이 화면은 빈 행조차 그리지 않고 헤더만 남긴다 — 빈 상태에서는 표 골격을 제거하는 것이 의도로 보임.
- 별도 주석 텍스트 없음.
