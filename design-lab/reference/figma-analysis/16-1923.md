# 16:1923 — 회원가입 - 닉네임 작성하기 (이름 중복 에러 상태)

## 역할 추정
닉네임 작성 화면의 에러 상태 변형. "확인"을 눌렀을 때 이름이 이미 사용 중이면 표시되는 검증 실패 화면. 16:1422(빈 상태), 16:2475(입력 중)와 같은 화면의 상태 3.

## 구성요소
16:1422와 동일한 레이아웃에 에러 메시지 추가:
- 우측 상단: "SIGN IN WITH GOOGLE" / 우측 최상단: 원형 "설정" 버튼
- 중앙 회색 패널
  - 제목: "What's your name?"
  - "이름 :" 입력창 — 빈 상태로 그려짐(에러 후 초기화된 모습으로 추정)
  - 이름 입력창 바로 아래: **빨간색 에러 텍스트 "이미 사용하고 있는 이름입니다"**
  - "분반 :" 입력창 — 빈 상태
  - 하단 "확인" 버튼

## 기능 목록
1. 이름 중복 검증 및 에러 표시
   - (a) 제출된 이름이 이미 존재하면 인라인 에러 메시지를 이름 필드 아래에 빨간색으로 표시
   - (b) 트리거: "확인" 클릭 시 서버(또는 DB) 중복 검사 실패
   - (c) mock: 금지 이름 배열(["test"] 등) 포함 여부로 판정. 에러 시 이 메시지 표시 + 재입력 허용. (와이어프레임처럼 입력을 비울지, 입력을 유지할지는 UX상 유지 쪽이 나음 — 원안은 비워진 모습)
2. 에러 해제
   - (a) 사용자가 이름을 수정하면 에러 메시지 제거
   - (b) 트리거: 이름 입력창 변경
   - (c) mock: onChange 시 error state 초기화
3. 나머지 기능은 16-1422.md와 동일

## 특이사항
- 닉네임 작성 3종 세트(16:1422 / 16:2475 / 16:1923)의 마지막 상태. 16:2475의 "test" 입력과 이어지는 시나리오(중복 이름 제출 → 에러)로 해석됨.
- 에러 메시지가 이름 필드에만 붙어 있음 — 분반 필드의 검증(빈 값 등)에 대한 와이어프레임은 없음. mock에서 빈 값 검증을 추가할지는 팀 판단.
- 에러 상태에서 입력창이 비어 있는 것이 의도(제출 후 클리어)인지 단순 와이어프레임 편의인지 불명확.
