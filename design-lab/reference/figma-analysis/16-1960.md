# 16:1960 — 전체 인원 관리

- 이미지: `design-lab/reference/figma/16-1960__전체_인원_관리.png`
- 캔버스 위치: (4827, 8500)

## 역할 추정
관리자 콘솔의 "전체 인원 관리" 페이지 기본 상태. 그룹 단위가 아니라 서비스 전체 가입 계정을 조회·삭제하는 화면. 사이드바 두 번째 메뉴("전체 인원 관리")로 진입.

## 구성요소
- 좌측 사이드바: "MADPUMP" / "그룹 관리" / "전체 인원 관리"(현재 페이지) / 하단 "로그아웃"
- 본문 타이틀: "전체 인원 관리" (그룹 관리와 달리 뒤로가기·탭 없음)
- 검색 줄: 라벨 "닉네임 검색" + 텍스트 입력창 + 돋보기 아이콘
- 테이블 (열): "닉네임" / "Google ID" / "가입일시" / "플레이 수" / "관리"
  - 행 3개: "박준서 / asdferq@google.com / 2026-07-02 19:00 / (빈칸) / 계정 삭제하기(빨간 링크)"
  - "이종혁", "라태형" 동일 패턴 + 빈 행 다수

## 기능 목록
1. 전체 회원 목록 조회
   - (a) 전체 가입자의 닉네임/Google ID/가입일시/플레이 수 표시
   - (b) 페이지 진입 시 자동 로드
   - (c) mock: 전역 users mock 배열 렌더
2. 닉네임 검색
   - (a) 닉네임으로 필터 (b) 입력 + 돋보기/엔터
   - (c) mock: includes 필터, 0건이면 "검색 결과 없음" 패턴(16:2151 참고) 재사용
3. 계정 삭제하기
   - (a) 특정 계정을 서비스에서 완전 삭제
   - (b) 행의 빨간 "계정 삭제하기" 클릭 → 확인 모달(16:2018)로 이어짐
   - (c) mock: 확인 모달 → 확인 시 배열에서 제거
4. 사이드바 내비게이션(그룹 관리로 이동) / 로그아웃

## 특이사항
- 그룹 관리의 "인원" 탭과 레이아웃·열이 거의 동일하나, 관리 액션이 "내보내기"(그룹 제외)가 아니라 "계정 삭제하기"(계정 자체 삭제), 그리고 "점수" 열과 탭·뒤로가기가 없음.
- 16:2018("전체 인원 관리 - 인원 삭제")이 이 화면 위에 삭제 확인 모달을 얹은 변형.
- 데이터는 placeholder (Google ID 전부 동일, 플레이 수 빈칸).
