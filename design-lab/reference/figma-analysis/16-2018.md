# 16:2018 — 전체 인원 관리 - 인원 삭제 (삭제 확인 모달)

- 이미지: `design-lab/reference/figma/16-2018__전체_인원_관리_-_인원_삭제.png`
- 캔버스 위치: (6393, 8500)

## 역할 추정
"전체 인원 관리"(16:1960)에서 특정 회원의 "계정 삭제하기"를 클릭했을 때 뜨는 삭제 사전 확인 모달 상태. 파괴적 작업에 대한 2단계 확인.

## 구성요소
- 배경: 16:1960과 동일 (사이드바 "MADPUMP/그룹 관리/전체 인원 관리/로그아웃", 타이틀 "전체 인원 관리", "닉네임 검색" 입력+돋보기, 테이블 닉네임/Google ID/가입일시/플레이 수/관리, 행 박준서·이종혁·라태형 + "계정 삭제하기" 빨간 링크)
- 중앙 확인 모달:
  - 문구: "**박준서**(asdferq@google.com) 님을 **삭제**하시겠습니까?" (닉네임과 '삭제'가 볼드, 닉네임+이메일을 함께 명시)
  - 버튼 2개: "확인"(어두운 배경, 좌측) / "취소"(회색 배경, 우측)

## 기능 목록
1. 계정 삭제 사전 확인
   - (a) 삭제 대상(닉네임+Google ID)을 명시하고 최종 확인/취소를 받음
   - (b) 트리거: 행의 "계정 삭제하기" 클릭
   - (c) mock: 대상 user를 state에 담아 모달 오픈, 문구에 nickname/email 보간
2. 삭제 확정 ("확인")
   - (a) 계정 삭제 실행 후 목록 갱신
   - (b) 모달의 "확인" 클릭
   - (c) mock: users 배열에서 제거 + 모달 닫기 (완료 알림 모달은 이 플로우 와이어프레임에 없음)
3. 삭제 취소 ("취소")
   - (b) "취소" 클릭 (c) mock: 모달만 닫기, 데이터 불변
4. (배경 기능) 닉네임 검색 / 사이드바 내비 / 로그아웃 — 16:1960과 동일

## 특이사항
- 16:1960의 모달 변형 프레임 (배경 내용 완전 동일).
- 매치 결과 삭제(17:749)는 "삭제되었습니다" 완료 알림형인데, 계정 삭제는 "삭제하시겠습니까?" 사전 확인형(확인/취소) — 두 삭제 UX 패턴이 다름. 구현 시 파괴적 작업은 사전 확인형으로 통일하는 것을 권장.
- 파괴적 액션인 "확인"이 강조(어두운) 버튼, "취소"가 회색 — 위치/강조 관례(보통 취소가 좌측 또는 무채색)와 함께 최종 디자인에서 재정의 가능.
