# 16:2111 — 그룹 관리 - 그룹 내 인원

이미지: `design-lab/reference/figma/16-2111__그룹_관리_-_그룹_내_인원.png`

## 역할 추정
그룹 관리자(운영자)용 관리 화면. 특정 그룹("몰입캠프 1분반")에 소속된 인원 목록을 조회/검색/내보내기(강퇴)하는 화면. 관리 플로우의 진입점 — 상단 탭 3개(인원 / 매치 결과 관리 / 랭킹 조회) 중 "인원" 탭이 활성(검정 배경)인 상태.

## 구성요소
- 좌측 사이드바 (회색, 고정)
  - 상단: "MADPUMP" 로고 텍스트
  - 메뉴: "그룹 관리", "전체 인원 관리"
  - 하단: "로그아웃"
- 본문 상단
  - "◀ 뒤로가기" (좌상단)
  - 그룹명 제목: "몰입캠프 1분반" (굵게)
  - 탭 버튼 3개: "인원"(활성/검정), "매치 결과 관리", "랭킹 조회"
- 검색 영역: 라벨 "닉네임 검색" + 텍스트 입력창 + 돋보기 아이콘 버튼
- 인원 테이블 (열 6개)
  - 헤더: 닉네임 | Google ID | 가입일시 | 플레이 수 | 점수 | 관리
  - 데이터 행 예시: "박준서 / asdferq@google.com / 2026-07-02 19:00 / (빈칸) / (빈칸) / 내보내기", 같은 형식으로 "이종혁", "라태형"
  - "관리" 열의 "내보내기"는 빨간 텍스트 링크
  - 나머지 행은 빈 행(placeholder)

## 기능 목록
1. 탭 전환 (인원 / 매치 결과 관리 / 랭킹 조회)
   - (a) 같은 그룹의 다른 관리 뷰로 이동
   - (b) 탭 버튼 클릭
   - (c) mock: 클라이언트 라우팅 or 탭 state 전환, 활성 탭 강조
2. 닉네임 검색
   - (a) 인원 목록을 닉네임으로 필터링
   - (b) 입력 후 돋보기 클릭(또는 Enter)
   - (c) mock: 배열 filter(includes) 즉시 필터
3. 인원 목록 조회
   - (a) 그룹 멤버의 닉네임/Google ID/가입일시/플레이 수/점수 표시
   - (b) 화면 진입 시 자동 로드
   - (c) mock: 고정 멤버 배열 렌더. 플레이 수/점수는 mock 매치 데이터에서 집계
4. 멤버 내보내기(강퇴)
   - (a) 그룹에서 멤버 제거
   - (b) 행의 "내보내기" 빨간 링크 클릭 → 확인 모달(별도 프레임 16:1780 존재)
   - (c) mock: confirm 모달 후 배열에서 제거, 토스트/재렌더
5. 뒤로가기
   - (a) 그룹 목록(그룹 관리 메인)으로 복귀
   - (b) "◀ 뒤로가기" 클릭
   - (c) mock: history.back 또는 그룹 목록 페이지로 이동
6. 사이드바 내비게이션 / 로그아웃
   - (a) 그룹 관리·전체 인원 관리 페이지 이동, 세션 종료
   - (b) 사이드바 항목 클릭
   - (c) mock: 라우팅, 로그아웃은 mock 세션 clear 후 메인으로

## 특이사항
- 같은 화면의 변형 프레임: 16:1780 "그룹 관리 - 그룹 내 인원 - 내보내기 버튼 눌렀을 때" = 이 화면 + 내보내기 확인 모달 오버레이. 즉 본 프레임이 base 상태.
- 관리 탭 공통 레이아웃(사이드바 + 뒤로가기 + 그룹명 + 탭 3개)이 17:530 계열과 완전히 동일 — 공용 관리 레이아웃 컴포넌트로 만들면 됨.
- "플레이 수", "점수" 열은 와이어프레임에 값이 비어 있음(집계 데이터 필요하다는 표시로 해석).
- Google ID 예시가 전 행 동일(asdferq@google.com) — placeholder임.
