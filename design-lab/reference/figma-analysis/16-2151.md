# 16:2151 — 그룹 관리 - 그룹 내 인원 (검색 결과 없음 상태)

- 이미지: `design-lab/reference/figma/16-2151__그룹_관리_-_그룹_내_인원.png`
- 캔버스 위치: (7667, 3350)

## 역할 추정
관리자용 "그룹 관리 → 특정 그룹(몰입캠프 1분반) → 인원" 탭에서 닉네임 검색 결과가 0건일 때의 빈 상태(empty state) 화면.

## 구성요소
- 좌측 사이드바: "MADPUMP" / "그룹 관리" / "전체 인원 관리" / 하단 "로그아웃"
- 본문 상단: "◀ 뒤로가기" / 타이틀 "몰입캠프 1분반"
- 탭 3개: "인원"(활성, 검정 배경) / "매치 결과 관리" / "랭킹 조회"
- 검색 줄: 라벨 "닉네임 검색" + 텍스트 입력창(빈 값) + 돋보기 아이콘
- 테이블 헤더만 표시 (열): "닉네임" / "Google ID" / "가입일시" / "플레이 수" / "관리"
- 테이블 본문 영역 중앙에 큰 굵은 텍스트: "검색 결과 없음"

## 기능 목록
1. 검색 결과 없음 빈 상태 표시
   - (a) 검색 결과 0건임을 명시적으로 알림 (행을 숨기고 안내 문구만)
   - (b) 닉네임 검색 실행 결과가 0건일 때
   - (c) mock: filteredList.length === 0 이면 테이블 바디 대신 "검색 결과 없음" 문구 렌더
2. 닉네임 검색
   - (b) 입력 + 돋보기 클릭/엔터 (c) mock: includes 필터
3. 탭 전환 / 뒤로가기 / 사이드바 내비 / 로그아웃

## 특이사항
- 같은 이름의 프레임이 3개 존재: 16:2111(기본 목록 상태, 타 담당), 16:2193(검색 1건 결과), 16:2151(본 프레임, 검색 0건). 즉 인원 탭의 상태 변형 시리즈 중 "결과 없음" 상태.
- 이 프레임의 테이블 헤더에는 "점수" 열이 없음 (닉네임/Google ID/가입일시/플레이 수/관리 5열). 반면 16:2193과 16:2062(랭킹 조회)에는 "점수" 열이 있는 6열 구성 — 와이어프레임 간 불일치이므로 구현 시 인원 탭 열 구성을 하나로 확정해야 함.
- 그룹 찾기 쪽에도 유사한 "해당하는 그룹명 없는 경우" 프레임(16:1893)이 있어, "검색 0건 → 빈 상태 문구" 패턴이 공통 컨벤션임.
