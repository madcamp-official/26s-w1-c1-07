# 16:2193 — 그룹 관리 - 그룹 내 인원 (검색 결과 1건 상태)

- 이미지: `design-lab/reference/figma/16-2193__그룹_관리_-_그룹_내_인원.png`
- 캔버스 위치: (6278, 3350)

## 역할 추정
관리자용 "그룹 관리 → 특정 그룹(몰입캠프 1분반) → 인원" 탭에서 닉네임 검색("박준서")을 실행해 결과 1건이 필터되어 표시된 상태.

## 구성요소
- 좌측 사이드바: "MADPUMP" / "그룹 관리" / "전체 인원 관리" / 하단 "로그아웃"
- 본문 상단: "◀ 뒤로가기" / 타이틀 "몰입캠프 1분반"
- 탭 3개: "인원"(활성, 검정 배경) / "매치 결과 관리" / "랭킹 조회"
- 검색 줄: 라벨 "닉네임 검색" + 입력창(값: "박준서") + 돋보기 아이콘
- 테이블 (열): "닉네임" / "Google ID" / "가입일시" / "플레이 수" / "점수" / "관리"
  - 결과 행 1개: "박준서 / asdferq@google.com / 2026-07-02 19:00 / (빈칸) / (빈칸) / 내보내기(빨간 링크)"
  - 그 아래 2행에 데이터 없이 "내보내기" 링크만 남아 있음 (와이어프레임 편집 잔재로 보임) + 빈 행 다수

## 기능 목록
1. 닉네임 검색 결과 표시
   - (a) 검색어와 일치하는 멤버만 필터해 표시
   - (b) 검색어 입력 + 돋보기 클릭/엔터
   - (c) mock: members.filter(m => m.nickname.includes(query))
2. 멤버 내보내기
   - (a) 그룹에서 멤버 제외 (b) 행의 빨간 "내보내기" 클릭
   - (c) mock: 확인 모달(16:1780 참고) → 배열 제거
3. 탭 전환 / 뒤로가기 / 사이드바 내비 / 로그아웃

## 특이사항
- 같은 이름 프레임 시리즈: 16:2111(기본 전체 목록) → 16:2193(본 프레임, 검색 1건) → 16:2151(검색 0건). 검색 상태 변형.
- 데이터 없는 2·3행에 "내보내기" 링크가 그대로 남아 있음 — 명백한 와이어프레임 잔재. 구현 시 데이터 있는 행에만 관리 버튼을 붙일 것.
- 이 프레임 테이블에는 "점수" 열이 포함(6열)되어 16:2062(랭킹 조회)와 동일 구성인 반면, 16:2151에는 점수 열이 없음(5열). 인원 탭의 정식 열 구성 확정 필요.
