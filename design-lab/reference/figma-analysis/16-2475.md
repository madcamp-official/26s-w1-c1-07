# 16:2475 — 회원가입 - 닉네임 작성하기 (입력 중 상태)

## 역할 추정
닉네임 작성 화면(16:1422)의 상태 변형. 이름 입력창에 값("test")이 입력된 "입력 중" 상태를 보여주는 프레임. 별도 화면이 아니라 동일 화면의 인터랙션 데모.

## 구성요소
16:1422와 동일한 레이아웃:
- 우측 상단: "SIGN IN WITH GOOGLE" / 우측 최상단: 원형 "설정" 버튼
- 중앙 회색 패널
  - 제목: "What's your name?"
  - "이름 :" 입력창 — **"test" 입력됨** (16:1422와의 유일한 차이)
  - "분반 :" 입력창 — 빈 상태
  - 하단 "확인" 버튼

## 기능 목록
16-1422.md와 동일 (이름 입력 / 분반 입력 / 확인 제출 / 헤더 공통). 이 프레임이 추가로 보여주는 것:
1. 입력값 표시
   - (a) 타이핑한 텍스트가 입력창에 표시됨
   - (b) 트리거: 키 입력
   - (c) mock: 기본 input 동작으로 충족. 별도 구현 불필요

## 특이사항
- 16:1422(빈 상태) → 16:2475(입력) → 16:1923(중복 에러)의 3단계 상태 시퀀스 중 가운데.
- 예시 이름이 "test"인데, 16:1923에서 "이미 사용하고 있는 이름입니다" 에러가 나오는 것과 이어지는 시나리오로 보임(= "test"라는 이름이 중복인 경우의 데모). mock에서 "test"를 중복 이름으로 하드코딩하면 와이어프레임 시나리오를 그대로 재현 가능.
- 구현 관점에서 이 프레임 자체는 새 기능을 정의하지 않음. 하나의 페이지 + 상태로 처리.
