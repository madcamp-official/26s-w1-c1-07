# 17:1019 — "게임1" (게임1 메인 플레이 화면)

이미지: `design-lab/reference/figma/17-1019__게임1.png`

## 역할 추정
게임1(숫자 맞추기 대결)의 인게임 메인 화면. 매칭 완료 후 실제 플레이가 진행되는 단계. 좌우에 두 플레이어 프로필, 중앙에 목표 숫자, 하단에 각자의 현재 숫자와 P/Q 조작 버튼이 있다. 유저 플로우상 로비/매칭 → **이 화면(플레이)** → 결과 화면 순으로 추정.

## 구성요소 (위치와 텍스트)
- 상단 중앙: 가로 바 형태 박스, 텍스트 **"(count down)"** — 라운드 제한시간 또는 시작 카운트다운 표시 영역
- 좌측 상단: 큰 사각 박스, 텍스트 **"프로필"** — P1 프로필 박스
- 우측 상단: 큰 사각 박스, 텍스트 **"프로필"** — P2 프로필 박스
- 중앙: 큰 사각 박스, 텍스트 **"타겟숫자"** — 두 플레이어가 맞춰야 하는 목표 숫자 표시
- 하단 좌측: 박스, 텍스트 **"P1 현재숫자"** — P1의 현재 숫자 값
- 하단 우측: 박스, 텍스트 **"P2 현재숫자"** — P2의 현재 숫자 값. 박스 위에 굵은 주석 텍스트 **"this is you"** (로컬 플레이어가 우측/P2 자리라는 표기)
- 하단 중앙: 원반(버튼 패드) 2개 — 왼쪽 원반에 **아래 방향 화살표 + 라벨 "P"**, 오른쪽 원반에 **위 방향 화살표 + 라벨 "Q"** — 키보드 P/Q 키에 대응하는 조작 표시

## 기능 목록
1. **카운트다운 타이머**
   - (a) 라운드 시작 전 3-2-1 또는 라운드 남은 시간 표시
   - (b) 트리거: 양측 매칭/준비 완료 시 자동 시작, 매 tick 갱신
   - (c) mock: `setInterval` 1초 tick, 0이 되면 라운드 종료 이벤트 발생
2. **타겟 숫자 제시**
   - (a) 라운드마다 목표 숫자를 중앙에 표시
   - (b) 트리거: 라운드 시작 시 서버(또는 mock 시드)에서 랜덤 생성
   - (c) mock: 시드 고정 랜덤으로 양쪽 클라이언트 동일 값 생성 또는 mock 서버 상태에서 push
3. **P 키 입력 = 현재 숫자 감소 (아래 화살표)**
   - (a) 내 현재 숫자를 감소시킴
   - (b) 트리거: 키보드 `P` keydown (또는 패드 클릭/탭)
   - (c) mock: keydown 리스너 → 로컬 상태 -1, 상대에게는 mock 채널로 반영
4. **Q 키 입력 = 현재 숫자 증가 (위 화살표)**
   - (a) 내 현재 숫자를 증가시킴
   - (b) 트리거: 키보드 `Q` keydown
   - (c) mock: keydown 리스너 → 로컬 상태 +1
5. **양 플레이어 현재 숫자 실시간 표시**
   - (a) P1/P2 각각의 현재 숫자를 하단 좌/우에 표시, 내 쪽에는 "this is you" 식 구분
   - (b) 트리거: 입력 이벤트마다 갱신, 상대 값은 네트워크 수신 시 갱신
   - (c) mock: 상대는 봇(랜덤 간격으로 타겟에 접근) 또는 BroadcastChannel로 두 탭 연결
6. **승패 판정**
   - (a) 타겟 숫자에 먼저 도달(또는 시간 종료 시 더 가까운 쪽)한 플레이어 승
   - (b) 트리거: 현재숫자 == 타겟숫자 도달 순간 또는 카운트다운 0
   - (c) mock: 상태 비교 후 결과 화면(오버레이)으로 전환
7. **프로필 표시**
   - (a) 양측 플레이어 정보(이름/아바타) 표시
   - (b) 트리거: 매칭 완료 시 데이터 수신
   - (c) mock: 하드코딩된 mock 유저 2명

## 특이사항
- "this is you" 주석은 로컬 플레이어 시점 구분이 필요함을 명시 — P2(우측)가 나. 실제 구현에서는 내 쪽을 시각적으로 강조해야 함.
- 게임1은 좌우 분할(수평 대칭) 레이아웃, 게임2(17:1234)는 상하 분할 레이아웃 — 게임별 화면 구도가 다름.
- P=감소(↓), Q=증가(↑) 매핑이 화살표로 명시됨.
- 담당 6개 프레임 중 "게임1"은 이 프레임 하나뿐이며 나머지 5개는 모두 "게임2" 계열.
