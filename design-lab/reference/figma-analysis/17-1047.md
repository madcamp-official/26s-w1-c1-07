# 17:1047 — 게임2 - P2

이미지: `design-lab/reference/figma/17-1047__게임2_-_P2.png`

## 역할 추정
게임2(낙하물 받기/떨어지는 오브젝트 게임)의 **P2 시점** 인게임 화면. 상단 P1 라인에서 오브젝트(붉은 액체가 담긴 캡슐/물약 모양)가 떨어지고, 하단 P2 라인 쪽에서 P/Q 버튼으로 좌우 이동하며 받거나 피하는 구조로 보임. 유저 플로우상 게임2 플레이 단계, 두 시점(P1/P2) 중 P2용 렌더링.

## 구성요소 (위치 포함)
- **상단 가로선 + "P1" 원형 배지**: 화면 상단을 가로지르는 라인 위 중앙-좌측에 P1 표시 원 — 상대(P1)의 위치/라인
- **중앙 낙하 오브젝트 3개**: 위에서 아래로 대각선 궤적을 그리며 떨어지는 캡슐 모양 오브젝트(하단이 빨간색) 3개 — 낙하 궤적/잔상 표현
- **중앙 하단 원반 버튼 2개 (반투명/비활성 톤)**:
  - 왼쪽 원반: 왼쪽 방향 화살표(←) + 라벨 "P"
  - 오른쪽 원반: 오른쪽 방향 화살표(→) + 라벨 "Q"
- **하단 가로선 + "P2" 원형 배지**: 화면 하단 라인 위 중앙-우측에 P2 표시 원 — 내(P2) 위치. P2 배지가 우측으로 이동해 있는 상태(이동 결과 표현)

## 기능 목록
1. **낙하 오브젝트 스폰/낙하 애니메이션**
   - (a) P1 위치에서 오브젝트가 생성되어 아래로 낙하
   - (b) 게임 루프(틱)마다 위치 갱신; P1의 위치/입력이 낙하 시작점 결정 추정
   - (c) mock: requestAnimationFrame 루프 + 오브젝트 배열 state, 상대는 봇
2. **P/Q 키로 좌우 이동**
   - (a) P=왼쪽, Q=오른쪽으로 자기 배지(P2)를 수평 이동
   - (b) 키보드 P/Q keydown(연타/홀드)
   - (c) mock: keydown 리스너로 x좌표 증감, 화면 경계 클램프
3. **상대(P1) 위치 실시간 표시**
   - (a) 상단 라인 위 P1 배지 위치를 상대 입력에 따라 갱신
   - (b) 네트워크 동기화(mock에서는 봇 AI)
   - (c) mock: 봇이 랜덤/추적 이동
4. **충돌(캐치) 판정**
   - (a) 낙하 오브젝트가 하단 라인 도달 시 P2 배지와의 겹침 판정 → 점수/데미지
   - (b) 오브젝트 y좌표가 하단 라인 도달 시
   - (c) mock: AABB/거리 기반 판정, 판정 결과에 따라 점수 state 갱신
5. **카운트다운 타이머** (이 프레임에는 없으나 쌍 프레임 17:1075에 존재)
   - (a) 라운드 제한시간 표시
   - (b) 게임 시작 시 자동
   - (c) mock: setInterval

## 특이사항
- **17:1364("게임2 - P2")와 픽셀 단위 완전 동일한 중복 프레임** (export 크기까지 16690 bytes로 동일). 다른 플로우 페이지에 복사된 사본으로 추정.
- **17:1075("게임2 - P1")와의 차이**: P1 프레임에는 상단에 "(count down)" 바가 있고, 오브젝트가 P쪽 원반 위로 떨어지며 Q 원반 위에 흐릿한(비활성) 오브젝트가 있음. 이 P2 프레임에는 카운트다운이 없고 오브젝트 궤적이 대각선 — 같은 게임의 다른 순간/다른 시점 스냅샷.
- P/Q 원반과 화살표가 반투명 처리됨 — 배경 힌트(입력 안내 오버레이)로 쓰인 듯. 실제 구현 시 키 안내 UI로 해석.
- 게임1과 달리 P/Q가 좌우 이동(←/→) 의미.
- 점수/체력 UI가 와이어프레임에 없음 — 구현 시 추가 필요 여부를 기능정의서와 대조해야 함.
