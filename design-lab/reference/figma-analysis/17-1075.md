# 17:1075 — 게임2 - P1

이미지: `design-lab/reference/figma/17-1075__게임2_-_P1.png`

## 역할 추정
게임2의 **P1 시점** 인게임 화면. P2 시점(17:1047/17:1364)과 같은 게임의 반대편 플레이어 뷰. 상단에 카운트다운이 있는 것으로 보아 이쪽이 더 완성된(정식) 인게임 레이아웃이고, P2 프레임은 다른 순간의 스냅샷일 수 있음.

## 구성요소 (위치 포함)
- **최상단 중앙**: "(count down)" 라벨 바 — 남은 시간 표시 영역
- **상단 가로선 + "P1" 원형 배지** (중앙-좌측): 내(P1) 위치 라인
- **중앙 낙하 오브젝트들**: 캡슐(하단 빨강) 오브젝트 3개가 P1 아래에서 수직으로 떨어져 P 원반 위에 도달해 있음. 오른쪽 Q 원반 위에는 **흐릿한(반투명 핑크) 캡슐 1개** — 아직 도달 전이거나 미스/고스트 상태 표현
- **중앙 반투명 원반 버튼 2개**: "P"(← 화살표), "Q"(→ 화살표)
- **하단 가로선 + "P2" 원형 배지** (중앙-우측): 상대(P2) 위치 라인

## 기능 목록
1. **카운트다운 타이머**
   - (a) 라운드 제한시간 표시/종료 처리
   - (b) 게임 시작 시 자동 시작, 0에서 승패 판정
   - (c) mock: setInterval + 종료 시 결과 전환
2. **오브젝트 낙하/스택 표현**
   - (a) P1 쪽에서 생성된 오브젝트가 낙하하며, P/Q 지점에 도달·누적되는 상태 표시
   - (b) 게임 틱마다 위치 갱신
   - (c) mock: rAF 루프, 도달 시 배열에서 제거+판정 이벤트
3. **P/Q 키 좌우 이동(또는 좌/우 슬롯 선택)**
   - (a) P=왼쪽, Q=오른쪽 입력으로 자기 배지 이동 혹은 좌/우 레인 선택
   - (b) keydown P/Q
   - (c) mock: keydown 리스너 + 레인/좌표 state
4. **상대(P2) 위치 실시간 표시**
   - (a) 하단 P2 배지 위치를 상대 입력으로 갱신
   - (b) 동기화 이벤트
   - (c) mock: 봇 이동
5. **판정 피드백(흐릿한 캡슐)**
   - (a) 미도달/예고/미스 상태의 오브젝트를 저채도로 구분 표시
   - (b) 판정 이벤트 시 상태 플래그 변경
   - (c) mock: 오브젝트 state(active/ghost)에 따른 opacity 분기

## 특이사항
- **17:1392("게임2 - P1")와 픽셀 단위 완전 동일한 중복** (20695 bytes 동일). 하나로 취급.
- **P2 프레임(17:1047)과의 차이점**: ① 상단 "(count down)" 바가 이 프레임에만 있음. ② 낙하 궤적이 P2 프레임은 대각선, 이 프레임은 수직으로 P 원반까지 도달+Q 원반 위 흐릿한 캡슐 존재. ③ 그 외 라인/배지/원반 배치는 동일.
- 카운트다운이 P1 프레임에만 있는 것은 누락으로 보임 — 실제 구현에서는 양 시점 모두 카운트다운 필요.
- P1/P2 시점 프레임 모두 "내 라인이 위/상대가 아래"가 아니라 P1이 항상 위 — 시점별 상하 반전을 하는지, 아니면 고정 배치에 시점 표시만 하는지는 결정 필요(와이어프레임상 고정 배치 + 자기 배지 인지 방식으로 추정).
