# 17:1107 — 게임2

이미지: `design-lab/reference/figma/17-1107__게임2.png`

## 역할 추정
게임2의 **컨셉/룰 설명 프레임**으로 추정. P1/P2 시점 프레임들과 달리 실제 UI 레이아웃이라기보다, 게임2의 아이디어(펜싱 대결 모티브 + 버튼 연타)를 설명하는 다이어그램에 가까움. 유저 플로우상으로는 게임2 진입 전 "게임 설명/튜토리얼" 화면이거나, 순수 기획 메모 프레임일 수 있음.

## 구성요소 (위치 포함)
- **상단 중앙-우측**: "변경" 텍스트가 든 원형 배지 — 이 게임(또는 버튼 역할)이 변경되었음을 알리는 주석으로 추정
- **상단 우측**: 원반 버튼 "P" 위로 아래 방향 화살표가 여러 겹(연타 표현) — P 버튼을 빠르게 연타하는 동작 도식
- **중앙 좌측**: 칼(에페/검)을 들고 찌르는 자세의 스틱 피겨 캐릭터 — 공격자(P1 추정)
- **중앙 우측**: 방패(부채꼴)를 들고 웅크린 스틱 피겨 캐릭터 — 방어자(P2 추정)
- **하단 전체 폭 큰 박스**: 내부 하단에 "(count down)" 텍스트 — 카운트다운/스테이지 바닥 영역

## 기능 목록
1. **연타 입력(P 버튼)**
   - (a) P 버튼을 연타해 공격(찌르기) 게이지/파워를 올리는 동작
   - (b) P keydown 연타
   - (c) mock: keydown 카운트 누적, 연타 속도(taps/sec) 계산
2. **공격/방어 역할 대결**
   - (a) 한쪽은 공격(찌르기), 한쪽은 방어(방패) — 비대칭 역할 or 라운드마다 교대 추정
   - (b) 라운드 시작 시 역할 배정
   - (c) mock: 역할 state + 봇 상대, 연타량 비교로 승패
3. **카운트다운**
   - (a) 라운드 제한시간
   - (b) 자동 시작
   - (c) mock: setInterval
4. **캐릭터 애니메이션(공격/방어 모션)**
   - (a) 연타/판정에 따라 캐릭터가 찌르기/막기 모션 표시
   - (b) 입력 이벤트/판정 이벤트
   - (c) mock: CSS 애니메이션/스프라이트 상태 전환

## 특이사항
- **"변경" 배지**: 이 프레임의 게임2 컨셉이 변경되었다는 기획 주석일 가능성이 높음. 낙하물 방식의 P1/P2 프레임들(17:1047/1075/1364/1392)이 변경 후(또는 전) 버전일 수 있음 → **게임2의 최종 룰이 "펜싱 연타"인지 "낙하물 받기"인지 기능정의서와 반드시 대조 필요.**
- 같은 이름 계열("게임2")이지만 P1/P2 시점 프레임들과 화면 구성이 완전히 다름 — 이 프레임은 인게임 HUD가 아니라 컨셉 도식.
- 이 프레임만 캔버스 위치 (-1809,6178)로 P1/P2 프레임들(-3486, y=6178)과 같은 행의 오른쪽에 있음 — 같은 기획 페이지 내 보조 설명 프레임.
- "(count down)" 텍스트가 바닥 박스 안에 있는 점도 다른 프레임(상단 바)과 다름 — 위치 확정 아님.
