# 17:1143 — "게임2" (게임 플레이 화면: 양측 Q 버튼 연타)

## 역할 추정
1:1 대결 게임의 **실제 플레이(연타) 화면**. 좌우 두 플레이어가 각각 자기 쪽 버튼(라벨 `Q`)을 연타해 공격(칼)과 방어(방패)를 겨루는 구도. 유저 플로우상 카운트다운(게임3 17:1424 계열) 직후의 본게임 단계로 추정.

## 구성요소 (위치와 함께)
- **좌상단**: 원기둥(아케이드/펌프 스타일) 버튼, 라벨 `Q`, 버튼 위에 **칼(검) 아이콘**이 꽂혀 있음 — 좌측(공격) 플레이어의 입력 버튼
- **우상단**: 동일한 원기둥 버튼, 라벨 `Q`, 버튼 위에 **방패 아이콘** — 우측(방어) 플레이어의 입력 버튼
- **중앙~하단 좌측**: 칼을 든 캐릭터(오른쪽으로 겨눔)
- **중앙~하단 우측**: 방패를 든 캐릭터(왼쪽을 방어)
- **하단**: 화면 폭 대부분의 큰 회색 박스(스테이지/바닥). 이 프레임에서는 박스 안 텍스트 없음(비어 있음)

## 기능 목록
1. **버튼 연타 입력 (좌/우 각각)**
   - (a) 각 플레이어가 자신의 버튼을 연타해 힘 게이지를 올림 (공격력/방어력)
   - (b) 트리거: 키보드 키(라벨상 `Q`; 상대는 `P`일 가능성) 또는 화면 버튼 탭
   - (c) mock: `keydown` 리스너로 키별 카운트 누적, 화면 버튼은 눌림 애니메이션과 함께 같은 카운터 증가
2. **공격/방어 대결 시각화**
   - (a) 연타량 차이에 따라 캐릭터 대치 상태(밀림/버팀)를 표시
   - (b) 트리거: 실시간 카운터 변화
   - (c) mock: 두 카운터 차에 비례해 캐릭터 x위치/기울기 이동, 임계치 도달 시 승패 판정
3. **하단 스테이지/정보 영역**
   - (a) 바닥 겸 상태 표시 영역(다른 변형 프레임에서는 카운트다운이 이 위치에 표기됨)
   - (b) 트리거: 게임 단계 전환
   - (c) mock: 단계별로 카운트다운/빈 상태/결과 텍스트를 이 영역에 조건부 렌더

## 특이사항
- **17:1460("게임3")과 픽셀 단위로 사실상 동일한 화면** — 이름만 다르고 내용 차이를 발견하지 못함. 복제 후 이름만 바꾼 것으로 추정.
- 17:1424("게임3")와 비교: 그 프레임은 상단이 `변경` 버튼 + `P` 버튼(화살표 강조)이고 하단에 `(count down)`이 있는 준비 단계 변형; 본 프레임은 양측 `Q` 버튼이 있는 플레이 단계.
- 두 버튼 모두 라벨이 `Q`인 점은 와이어프레임 단계의 placeholder로 보임(실제로는 좌 Q / 우 P처럼 분리 필요). mock에서는 좌우 키를 다르게 배정할 것.
