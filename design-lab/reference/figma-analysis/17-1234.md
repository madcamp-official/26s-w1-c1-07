# 17:1234 — "게임2 - P1" (게임2 P1 시점 노트 낙하 화면)

이미지: `design-lab/reference/figma/17-1234__게임2_-_P1.png`

## 역할 추정
게임2(리듬/노트 낙하형 대결)의 인게임 화면 중 **P1 시점(상단 영역)**을 설명하는 프레임. 화면이 수평선 2개로 상/하 두 영역으로 나뉘며, 상단이 P1 구역, 하단이 P2 구역인 상하 분할 대전 레이아웃. 유저 플로우상 매칭 후 게임2 플레이 단계.

## 구성요소 (위치와 텍스트)
- 최상단 중앙: 가로 바 박스, 텍스트 **"(count down)"**
- 상단: 가로 구분선(수평선). 선 중앙에 원형 배지 **"P1"** — 상단 영역이 P1 구역임을 표시
- P1 구역 중앙: **낙하하는 노트(캡슐) 오브젝트 3개**가 세로로 이어짐 — 위쪽은 연하고 아래로 갈수록 붉은색이 진해짐(낙하 진행/판정 근접 표현으로 추정). 노트들은 왼쪽 원반(P 패드) 방향으로 떨어짐
- P1 구역 중앙~하단: 원반(패드) 2개 나란히 —
  - 왼쪽 원반: 라벨 **"P"**, 원반 뒤에 **왼쪽 방향 큰 화살표**(반투명) — 노트가 이 패드로 유도됨 / 방향 지시
  - 오른쪽 원반: 라벨 **"Q"**, 그 위에 연한(비활성) 노트 1개
- 하단: 두 번째 가로 구분선. 선 우측에 원형 배지 **"P2"** — 하단 영역이 P2 구역
- P2 구역: 비어 있음 (이 프레임은 P1 쪽 동작만 도식화)

## 기능 목록
1. **카운트다운 타이머**
   - (a) 게임 시작/라운드 남은 시간 표시
   - (b) 트리거: 게임 시작 시 자동
   - (c) mock: setInterval tick, 종료 시 결과 전환
2. **노트 낙하(스폰 및 이동)**
   - (a) 노트가 위에서 P 또는 Q 패드 레인으로 떨어짐
   - (b) 트리거: 게임 루프가 일정/패턴 간격으로 스폰, requestAnimationFrame으로 낙하
   - (c) mock: 시드 고정 패턴 배열 + rAF 애니메이션, 패드 근접 시 판정선 진입
3. **P/Q 키 판정 입력**
   - (a) 노트가 해당 패드에 도달하는 타이밍에 P 또는 Q 키를 눌러 히트 판정
   - (b) 트리거: keydown `P`/`Q`, 노트 위치와의 거리로 hit/miss 판정
   - (c) mock: 판정 윈도우(예: ±150ms) 내 입력이면 hit, 점수 가산; miss 시 감점/게이지 감소
4. **판정 근접 시각화 (노트 색 변화)**
   - (a) 노트가 판정선에 가까울수록 붉은 채움이 진해짐
   - (b) 트리거: 낙하 진행률에 따라 자동
   - (c) mock: 진행률 기반 CSS/캔버스 색 보간
5. **상하 분할 P1/P2 동시 표시**
   - (a) 내 구역과 상대 구역을 한 화면에서 동시에 보여줌 (상대 진행 상황 관전)
   - (b) 트리거: 상대 입력/노트 상태를 실시간 수신
   - (c) mock: 봇이 확률적으로 hit/miss 하는 상대 시뮬레이션, 또는 BroadcastChannel 두 탭 연동
6. **승패 판정**
   - (a) 카운트다운 종료 시 점수(또는 체력) 비교로 승자 결정
   - (b) 트리거: 타이머 0
   - (c) mock: 점수 상태 비교 → 결과 오버레이

## 특이사항
- 프레임 이름의 "- P1" 접미사가 시사하듯, 이 프레임은 P1 관점의 동작(노트가 P 패드로 떨어지는 순간)을 설명하는 상태 스냅샷. P2 구역은 의도적으로 빈 상태.
- 왼쪽 화살표는 "이번 노트는 P(왼쪽) 패드" 라는 방향 지시로 추정 — 노트마다 P/Q 레인이 지정됨.
- 같은 "게임2" 이름의 프레임들(16:1573/17:1266 펜싱 도식, 16:1607/17:1302 Q 패드 도식)과 화면 구도가 완전히 다름. 이 프레임은 상하 분할 리듬형, 나머지는 펜싱 대결형 도식 — 게임2 컨셉이 (i) 노트 낙하 조작 UI와 (ii) 펜싱 캐릭터 연출의 결합이거나, 서로 다른 버전의 게임2 안일 가능성이 있음. 구현 전 어느 안이 확정인지 확인 필요.
