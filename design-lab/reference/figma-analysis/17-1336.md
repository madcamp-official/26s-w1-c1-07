# 17:1336 — 게임1

이미지: `design-lab/reference/figma/17-1336__게임1.png`

## 역할 추정
게임1(펌핑/숫자 맞추기 대결)의 인게임 플레이 화면. 매칭 완료 후 실제 대결이 진행되는 메인 게임 화면으로, 유저 플로우상 "대기실/매칭 → **게임1 플레이** → 결과" 단계에 해당. 두 플레이어가 P/Q 버튼으로 자신의 "현재숫자"를 조작해 중앙의 "타겟숫자"에 맞추는(또는 도달하는) 구조로 보임.

## 구성요소 (위치 포함)
- **좌상단**: "프로필" 박스 (P1 플레이어 프로필 영역)
- **우상단**: "프로필" 박스 (P2 플레이어 프로필 영역)
- **상단 중앙**: "(count down)" 라벨 바 — 남은 시간 카운트다운 표시 영역
- **중앙 대형 박스**: "타겟숫자" — 이번 라운드에 맞춰야 하는 목표 숫자 표시
- **좌하단 박스**: "P1 현재숫자" — P1의 현재 값 표시
- **우하단 박스**: "P2 현재숫자" — P2의 현재 값 표시. 박스 위에 굵은 주석 텍스트 "**this is you**" → 이 화면은 P2 시점(내 숫자는 오른쪽)
- **하단 중앙**: 물리 버튼 모양의 원반 2개
  - 왼쪽 원반: 아래 방향 화살표 + 라벨 "P" (값 감소 추정 — 화살표 방향상 감소/펌프다운)
  - 오른쪽 원반: 위 방향 화살표 + 라벨 "Q" (값 증가 추정)

## 기능 목록
1. **카운트다운 타이머**
   - (a) 라운드 제한시간을 실시간 표시, 0이 되면 라운드/게임 종료
   - (b) 게임 시작 시 자동 시작
   - (c) mock: `setInterval` 기반 로컬 타이머, 종료 시 결과 화면 전환
2. **타겟숫자 표시**
   - (a) 두 플레이어가 도달해야 할 목표 숫자를 크게 표시
   - (b) 라운드 시작 시 서버(또는 mock 시드)에서 생성
   - (c) mock: 시드 고정 랜덤 정수 생성, 양쪽 클라이언트 동일 값
3. **P/Q 키 입력으로 현재숫자 조작**
   - (a) P 키(감소·↓), Q 키(증가·↑)로 자신의 현재숫자를 변경 — 연타 게임 코어
   - (b) 키보드 P/Q keydown 또는 화면 버튼 탭
   - (c) mock: keydown 리스너 + 온스크린 버튼, 로컬 state 갱신
4. **양측 현재숫자 실시간 표시**
   - (a) P1/P2의 현재숫자를 각각 표시, 상대 값도 실시간 동기화
   - (b) 키 입력마다 갱신
   - (c) mock: 상대는 봇(랜덤 간격 자동 증가)으로 시뮬레이션
5. **프로필 표시**
   - (a) 양 플레이어의 닉네임/아바타 표시
   - (b) 매칭 시 전달받은 데이터
   - (c) mock: 하드코딩된 mock 유저 2명
6. **승패 판정** (화면에 명시는 없으나 필수)
   - (a) 타겟숫자에 먼저 도달(또는 시간 종료 시 더 근접)한 쪽 승리
   - (b) 현재숫자 == 타겟숫자 도달 or 타이머 0
   - (c) mock: 로컬 비교 후 결과 화면 전환

## 특이사항
- "this is you" 주석은 P2 시점 화면임을 나타내는 와이어프레임 주석 — 실제 구현에서는 "내" 숫자를 구분 표시해야 함(자기 쪽 하이라이트).
- P/Q 원반 버튼 모티브는 게임2 프레임들(17:1047, 17:1075, 17:1107)에도 반복 등장 — 공용 입력 컴포넌트.
- 게임1은 이 한 프레임뿐이며 게임2처럼 P1/P2 시점 분리 프레임이 없음 → 좌우 반전 없이 단일 레이아웃으로 P1/P2를 표기하는 방식으로 추정.
- 화살표 방향(P=↓, Q=↑)이 게임2 프레임(P=←, Q=→)과 다름: 게임별로 키 의미가 달라짐.
