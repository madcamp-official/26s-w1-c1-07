# 17:1424 — "게임3" (베팅/포인트 변경 단계)

## 역할 추정
1:1 대결 게임 화면 중 **라운드 시작 전 준비/베팅 단계**로 추정. 하단에 `(count down)` 텍스트가 있어, 카운트다운이 진행되는 동안 플레이어가 무언가(베팅 포인트 등)를 "변경"할 수 있는 상태를 표현. 유저 플로우상 매칭 완료 → (이 화면: 카운트다운 + 변경 기회) → 실제 게임 플레이 순서로 위치할 가능성이 높다.

## 구성요소 (위치와 함께)
- **상단 중앙~우측**:
  - 원형 버튼, 텍스트 `변경` (상단 중앙 부근)
  - 그 오른쪽에 **버튼(펌프 버튼) 모양 아이콘 + 아래로 향하는 화살표 2개**(누르라는 표시), 아이콘 위 라벨 `P`
- **중앙~하단**: 좌우 대치 구도의 캐릭터 2명
  - 좌측 캐릭터: 칼(검)을 들고 오른쪽을 겨눔 (공격 측)
  - 우측 캐릭터: 방패를 들고 왼쪽을 방어 (방어 측)
- **하단**: 화면 폭 대부분을 차지하는 큰 회색 박스(스테이지/바닥 겸 오버레이 영역), 그 안에 텍스트 `(count down)`

## 기능 목록
1. **카운트다운 표시**
   - (a) 라운드 시작까지 남은 시간을 표시
   - (b) 트리거: 화면 진입(매칭 완료/라운드 전환) 시 자동 시작
   - (c) mock: `setInterval` 기반 3-2-1 카운트다운 숫자 표시 후 게임 상태로 전환
2. **변경 버튼**
   - (a) 카운트다운 동안 선택(베팅 포인트, 역할/아이템 등)을 변경하는 기능으로 추정
   - (b) 트리거: `변경` 원형 버튼 클릭
   - (c) mock: 클릭 시 선택 변경 모달/토글, 카운트다운 종료 시 비활성화
3. **P 버튼(펌프 입력) 안내**
   - (a) `P` 라벨이 붙은 물리 버튼(키)을 누르라는 시각 안내 — 게임의 핵심 입력(펌프 연타)이 P 키임을 표시
   - (b) 트리거: 게임 중 키 입력(P 키 또는 화면 버튼 탭)
   - (c) mock: `keydown('p')` 리스너 + 모바일용 화면 탭 버튼, 누를 때마다 게이지 증가
4. **공격/방어 캐릭터 상태 표시**
   - (a) 두 플레이어의 역할(칼=공격, 방패=방어)과 대치 상태를 시각화
   - (b) 트리거: 서버/게임 상태에 따라 렌더
   - (c) mock: 좌=나, 우=상대 고정 배치, 역할 상태값에 따라 스프라이트 교체

## 특이사항
- 같은 이름 "게임3" 프레임이 17:1460에도 존재하나, 17:1460은 17:1143("게임2")과 사실상 동일한 내용(Q 버튼 2개 + 캐릭터)이고, 이 프레임(17:1424)만 `변경` 버튼 + `P` 버튼 + `(count down)`이 있는 변형이다.
- `(count down)`은 괄호 표기로 보아 실제 UI 텍스트가 아니라 "여기에 카운트다운이 들어간다"는 와이어프레임 주석.
- 버튼 라벨 `P`와 `Q`(다른 프레임)는 키보드 키 할당(플레이어별 입력 키)일 가능성 있음 — 게임2/3 프레임에서 좌측 버튼은 칼, 우측 버튼은 방패 아이콘이 올라가 있어 각자 자신의 키를 연타하는 구조로 추정.
