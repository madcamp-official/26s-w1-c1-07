# 17:530 — 그룹 관리 - 매치 결과 관리

이미지: `design-lab/reference/figma/17-530__그룹_관리_-_매치_결과_관리.png`

## 역할 추정
그룹 관리자용 매치 결과 관리 화면의 base 상태. "매치 결과 관리" 탭이 활성(검정)이며, 그룹 내에서 벌어진 1:1 매치들의 결과 목록을 기간 필터로 조회하고, 각 매치의 결과를 수정/삭제할 수 있다. 이후 프레임들(17:585 수정 모달 → 17:637 수정 완료 → 17:695 삭제 모달, 17:826 수정이력)의 출발점.

## 구성요소
- 좌측 사이드바: "MADPUMP" / "그룹 관리" / "전체 인원 관리" / "로그아웃" (16:2111과 동일)
- 본문 상단: "◀ 뒤로가기", 그룹명 "몰입캠프 1분반", 탭 3개 "인원" / "매치 결과 관리"(활성) / "랭킹 조회"
- 필터 영역: 깔때기(필터) 아이콘 + 날짜 입력창 2개("~"로 연결된 기간 범위) + 돋보기(검색 실행) 아이콘
- 매치 결과 테이블 (열 5개)
  - 헤더: 매치 일시 | Player 1 | Player 2 | 매치 결과 | 관리
  - 행 예시:
    - "2026-07-03 19:00 / 박준서 / 이종혁 / Player 1 Win / 결과수정 삭제"
    - "2026-07-03 12:00 / 라태형 / 허서준 / Player 2 Win / 결과수정 삭제"
    - "2026-07-02 20:00 / 박준서 / 라태형 / DRAW / 결과수정 삭제"
  - "관리" 열의 "결과수정", "삭제"는 빨간 텍스트 링크 2개
  - 나머지는 빈 행
- 우하단 버튼: "매치 결과 수정이력 조회" (회색 버튼)

## 기능 목록
1. 기간 필터 조회
   - (a) 매치 일시 범위(시작~끝)로 목록 필터
   - (b) 두 날짜 입력 후 돋보기 클릭
   - (c) mock: date input 2개, 배열 filter로 범위 비교
2. 매치 결과 목록 조회
   - (a) 매치 일시/양 플레이어/결과(Player 1 Win, Player 2 Win, DRAW) 표시, 최신순 정렬로 보임
   - (b) 화면 진입 시
   - (c) mock: 매치 mock 배열 렌더, 일시 내림차순 sort
3. 결과 수정
   - (a) 특정 매치의 결과 값을 변경
   - (b) 행의 "결과수정" 클릭 → 수정 모달(17:585) 오픈
   - (c) mock: 선택 매치를 state로 넘겨 모달 오픈
4. 결과 삭제
   - (a) 매치 기록 삭제
   - (b) 행의 "삭제" 클릭 → 삭제 확인 모달(17:695) 오픈
   - (c) mock: confirm 모달 후 배열에서 제거
5. 매치 결과 수정이력 조회
   - (a) 수정된 매치들의 변경 전/후 이력 페이지(17:826)로 이동
   - (b) 우하단 "매치 결과 수정이력 조회" 버튼 클릭
   - (c) mock: 수정 시마다 이력 배열에 push해 두고 별도 뷰로 라우팅
6. 탭 전환 / 뒤로가기 / 사이드바 / 로그아웃 — 16:2111과 동일

## 특이사항
- 이 프레임을 base로 하는 변형 3개: 17:585(수정 모달), 17:637(수정 완료 확인 모달), 17:695(삭제 확인 모달). 배경 테이블 데이터가 모두 동일.
- 매치 결과 표기는 "Player 1 Win" / "Player 2 Win" / "DRAW" 3가지 — 결과 enum이 무승부 포함 3값임이 확정됨.
- 필터가 인원 탭(닉네임 검색)과 달리 날짜 범위 필터 — 탭마다 필터 UI가 다름.
