# 17:585 — 그룹 관리 - 매치 결과 관리 - 결과 수정

이미지: `design-lab/reference/figma/17-585__그룹_관리_-_매치_결과_관리_-_결과_수정.png`

## 역할 추정
17:530(매치 결과 관리)에서 특정 행의 "결과수정"을 클릭했을 때 뜨는 결과 수정 모달 상태. 관리자가 매치 결과 값을 3택으로 다시 선택하는 단계. 다음 단계는 17:637(수정 완료 확인).

## 구성요소
- 배경: 17:530과 동일한 매치 결과 관리 화면(사이드바, 탭, 기간 필터, 테이블, 우하단 "매치 결과 수정이력 조회" 버튼)이 모달 뒤에 그대로 보임 (dim 처리 없음 — 와이어프레임 한계로 보임)
- 중앙 모달 (흰 배경, 테두리)
  - 제목: "매치 결과 수정" (굵게, 중앙)
  - 매치 일시: "2026-07-03 19:00"
  - 대진 표기: "Player 1 (박준서) vs Player 2 (이종혁)" (닉네임 굵게)
  - 결과 선택 버튼 3개 (가로 배치, 회색): "Player 1 Win" | "DRAW" | "Player 2 Win"
  - 닫기(X)나 취소 버튼은 와이어프레임에 보이지 않음

## 기능 목록
1. 수정 대상 매치 정보 표시
   - (a) 어떤 매치를 수정 중인지(일시, 두 플레이어) 보여줌
   - (b) "결과수정" 클릭으로 모달 오픈 시
   - (c) mock: 선택된 매치 객체를 모달 props로 전달
2. 결과 선택 (3택)
   - (a) 새 결과 값(Player 1 Win / DRAW / Player 2 Win) 지정
   - (b) 세 버튼 중 하나 클릭
   - (c) mock: 클릭 즉시 매치 결과 갱신 + 수정이력 push(이전값→새값, 수정일시) + 확인 모달(17:637)로 전환. 별도 "저장" 버튼이 없으므로 버튼 클릭 = 확정으로 해석
3. 모달 닫기/취소
   - (a) 수정 없이 복귀
   - (b) 와이어프레임에 명시된 UI 없음 — 오버레이 클릭/ESC로 구현 권장
   - (c) mock: backdrop 클릭 시 close

## 특이사항
- 17:530의 오버레이 변형. 배경 데이터 동일.
- 취소/닫기 버튼이 없다는 점이 삭제 모달(17:695 — 삭제/취소 버튼 있음)과 대비됨. 실제 구현 시 취소 수단 추가 필요(주석 없음, 판단 필요 사항).
- 현재 결과 값이 어떤 버튼인지 강조 표시가 없음 — 구현 시 현재값 하이라이트를 넣을지 결정 필요.
- 선택 후 상태는 17:637 "결과 수정 확인" 프레임이 이어짐("수정되었습니다." 메시지) — 즉 선택 즉시 저장되는 플로우.
