# 17:637 — 그룹 관리 - 매치 결과 관리 - 결과 수정 확인

이미지: `design-lab/reference/figma/17-637__그룹_관리_-_매치_결과_관리_-_결과_수정_확인.png`

## 역할 추정
17:585(결과 수정 모달)에서 결과 버튼을 누른 직후의 완료 확인 모달. "수정되었습니다."라는 완료 메시지와 최종 결과 값을 보여주고 "확인"으로 닫는다. 수정 플로우의 마지막 단계.

## 구성요소
- 배경: 17:530과 동일한 매치 결과 관리 화면
- 중앙 모달
  - 제목: "매치 결과 수정" (굵게)
  - 매치 일시: "2026-07-03 19:00"
  - 대진: "Player 1 (박준서) vs Player 2 (이종혁)"
  - 확정된 결과: "Player 1 Win" (굵게, 크게, 중앙)
  - 안내 문구: "수정되었습니다."
  - 버튼: "확인" (회색, 중앙 1개)

## 기능 목록
1. 수정 완료 피드백
   - (a) 어떤 매치가 어떤 결과로 수정 확정됐는지 표시
   - (b) 17:585에서 결과 버튼 클릭 직후 자동 표시
   - (c) mock: 모달 내용만 교체(수정 모달 → 완료 모달), 결과 값은 방금 선택한 값
2. 확인(닫기)
   - (a) 모달을 닫고 갱신된 테이블로 복귀
   - (b) "확인" 버튼 클릭
   - (c) mock: 모달 close + 테이블 재렌더(이미 갱신된 state 반영)

## 특이사항
- 17:585와 같은 모달 프레임의 후속 상태(2단계 플로우: 선택 → 완료 확인). 배경 동일.
- 예시에서 배경 테이블의 해당 행 결과도 "Player 1 Win"으로 원래 값과 같아, 와이어프레임상으로는 "선택한 값 그대로 확정 표시"라는 의도만 확인됨.
- 이 완료 모달 패턴("수정되었습니다." + 확인)은 삭제 플로우에는 대응 프레임이 없음 — 삭제는 확인 모달(17:695)만 존재. 삭제 완료 피드백은 토스트 등으로 자체 판단 필요.
