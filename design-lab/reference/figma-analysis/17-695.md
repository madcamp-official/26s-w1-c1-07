# 17:695 — 그룹 관리 - 매치 결과 관리 - 결과 삭제

이미지: `design-lab/reference/figma/17-695__그룹_관리_-_매치_결과_관리_-_결과_삭제.png`

## 역할 추정
17:530(매치 결과 관리)에서 행의 "삭제"를 클릭했을 때 뜨는 삭제 확인 모달. 파괴적 동작이므로 삭제(빨강)/취소 2버튼 확인 단계를 둔다.

## 구성요소
- 배경: 17:530과 동일한 매치 결과 관리 화면
- 중앙 모달
  - 제목: "매치 결과 삭제" (굵게)
  - 매치 일시: "2026-07-03 19:00"
  - 대진: "Player 1 (박준서) vs Player 2 (이종혁)"
  - 현재 결과: "Player 1 Win" (굵게, 크게)
  - 확인 문구: "매치 결과를 삭제하시겠습니까?"
  - 버튼 2개 (가로): "삭제" (빨간 배경, 흰 글씨) | "취소" (회색)

## 기능 목록
1. 삭제 대상 매치 정보 표시
   - (a) 삭제할 매치의 일시/대진/현재 결과를 보여줘 오조작 방지
   - (b) 목록의 "삭제" 링크 클릭 시
   - (c) mock: 선택 매치 객체를 모달 props로 전달
2. 삭제 확정
   - (a) 매치 기록을 목록에서 제거 (점수/랭킹 재계산에 영향)
   - (b) 빨간 "삭제" 버튼 클릭
   - (c) mock: 배열에서 splice, 모달 close, 필요시 랭킹/플레이 수 mock 재집계. 완료 모달 프레임은 없으므로 토스트 정도로 처리
3. 취소
   - (a) 삭제 없이 모달 닫기
   - (b) "취소" 버튼 클릭
   - (c) mock: 모달 close만

## 특이사항
- 17:530의 오버레이 변형. 배경 데이터 동일.
- 유일하게 빨간(위험) 버튼을 쓰는 프레임 — 파괴적 액션의 시각 규칙 근거.
- 수정 플로우(17:585→17:637)와 달리 삭제 완료 확인 프레임은 없음.
- 삭제된 매치가 "매치 결과 수정이력"(17:826)에 남는지는 와이어프레임에 정의 없음 — 스펙 결정 필요.
