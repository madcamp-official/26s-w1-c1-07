# 17:749 — 그룹 관리 - 매치 결과 관리 - 결과 삭제 확인

- 이미지: `design-lab/reference/figma/17-749__그룹_관리_-_매치_결과_관리_-_결과_삭제_확인.png`
- 캔버스 위치: (10757, 5435)

## 역할 추정
관리자(그룹 관리)용 화면. "그룹 관리 → 특정 그룹(몰입캠프 1분반) → 매치 결과 관리" 탭에서 특정 매치의 "삭제"를 실행한 직후, 삭제 완료를 알리는 확인 모달이 떠 있는 상태. 삭제 플로우의 마지막 단계(완료 알림)로 보인다. (이름은 "삭제 확인"이지만 모달 문구가 과거형 "삭제되었습니다"이므로 확인 다이얼로그가 아니라 완료 알림임.)

## 구성요소
- 좌측 사이드바 (회색, 세로 전체):
  - 상단 로고 텍스트 "MADPUMP"
  - 메뉴 "그룹 관리"
  - 메뉴 "전체 인원 관리"
  - 하단 "로그아웃"
- 본문 상단:
  - "◀ 뒤로가기" (아이콘+텍스트)
  - 그룹명 타이틀 "몰입캠프 1분반"
  - 탭 3개: "인원" / "매치 결과 관리"(활성, 검정 배경) / "랭킹 조회"
- 필터 줄: 필터(깔때기) 아이콘 + 날짜 입력창 2개(`___ ~ ___`, 기간 범위) + 돋보기(검색) 아이콘
- 매치 결과 테이블 (열): "매치 일시" / "Player 1" / "Player 2" / "매치 결과" / "관리"
  - 데이터 행 예: "2026-07-03 19:00 | 박준서 | 이종혁 | Player 1 Win | 결과수정 삭제(빨간 링크)"
  - 그 외 "2026-07-03 12:0…", "2026-07-02 20:0…" 행 + 빈 행 다수
- 중앙 모달:
  - 문구 "매치 결과가 삭제되었습니다."
  - 버튼 "확인"
- 우측 하단 버튼: "매치 결과 수정이력 조회"

## 기능 목록
1. 매치 결과 삭제 완료 알림 모달
   - (a) 삭제가 완료되었음을 알리고 목록으로 복귀
   - (b) 트리거: 관리 열의 "삭제" 클릭 → (삭제 실행) → 이 모달 표시. "확인" 클릭 시 모달 닫힘
   - (c) mock: 삭제 클릭 시 로컬 상태에서 해당 행 제거 후 alert 스타일 모달 표시, 확인 시 닫기
2. 매치 결과 삭제 ("삭제" 링크)
   - (a) 특정 매치 기록 제거
   - (b) 행별 빨간 "삭제" 텍스트 클릭
   - (c) mock: 배열 filter로 행 제거 (별도 사전 확인 다이얼로그가 와이어프레임엔 없음 — 완료 모달만 존재)
3. 매치 결과 수정 ("결과수정" 링크)
   - (a) 매치 결과(승패 등) 수정 진입
   - (b) 행별 빨간 "결과수정" 클릭
   - (c) mock: 인라인 편집 또는 수정 모달 (별도 프레임에 수정 UI가 있을 것으로 추정)
4. 기간 필터 검색
   - (a) 매치 일시 범위(시작~끝)로 목록 필터
   - (b) 날짜 2개 입력 후 돋보기 클릭
   - (c) mock: 두 date input + 배열 filter
5. 탭 전환 (인원 / 매치 결과 관리 / 랭킹 조회)
   - (b) 탭 버튼 클릭 (c) mock: 클라이언트 라우팅 또는 탭 state
6. 매치 결과 수정이력 조회
   - (a) 결과수정/삭제 이력 감사 로그 열람
   - (b) 우하단 버튼 클릭
   - (c) mock: 수정이력 목록 모달/페이지, mock 이력 배열
7. 사이드바 내비게이션(그룹 관리/전체 인원 관리), 뒤로가기, 로그아웃

## 특이사항
- 배경 화면은 "매치 결과 관리" 기본 프레임(다른 담당자 분석분일 가능성)과 동일하고, 모달만 얹힌 변형.
- 모달 문구가 과거형 → 삭제 전 확인(정말 삭제하시겠습니까?)이 아니라 삭제 후 완료 알림. 전체 인원 관리의 삭제(16:2018)는 사전 확인형(확인/취소)인 것과 대비됨 — 구현 시 두 패턴을 통일할지 결정 필요.
- "매치 결과 수정이력 조회" 버튼은 이 탭 고유 요소.
