# 17:826 — 그룹 관리 - 매치 결과 관리 - 매치결과 수정이력

이미지: `design-lab/reference/figma/17-826__그룹_관리_-_매치_결과_관리_-_매치결과_수정이력.png`

## 역할 추정
17:530에서 우하단 "매치 결과 수정이력 조회" 버튼으로 진입하는 별도 페이지(모달이 아님). 관리자가 수정한 매치 결과들의 변경 전/후 감사 로그(audit log)를 기간 필터로 조회한다. 탭 3개가 없고 제목 아래 "매치 결과 수정 이력" 부제가 있는 독립 서브페이지.

## 구성요소
- 좌측 사이드바: "MADPUMP" / "그룹 관리" / "전체 인원 관리" / "로그아웃" (공통)
- 본문 상단: "◀ 뒤로가기", 그룹명 "몰입캠프 1분반", 부제 "매치 결과 수정 이력" (굵게, 작게)
  - 탭 버튼 3개는 없음
- 필터 영역: 깔때기 아이콘 + 날짜 입력창 2개("~" 범위) + 돋보기 아이콘 (17:530과 동일 패턴)
- 수정이력 테이블 (열 6개)
  - 헤더: 매치 일시 | Player 1 | Player 2 | 이전 결과 | 이후 결과 | 수정 일시
  - 행 예시: "2026-07-03 19:00 / 박준서 / 이종혁 / Player 1 Win / DRAW / 2026-07-09 23:00"
  - 나머지 빈 행. 관리(수정/삭제) 열 없음 — 읽기 전용
- 우하단 버튼 없음

## 기능 목록
1. 수정이력 목록 조회
   - (a) 수정된 매치별로 이전 결과→이후 결과와 수정 일시를 표시 (읽기 전용 감사 로그)
   - (b) 17:530의 "매치 결과 수정이력 조회" 버튼으로 진입 시 자동 로드
   - (c) mock: 결과 수정 시마다 {matchAt, p1, p2, before, after, editedAt}을 이력 배열에 push, 이 뷰에서 렌더
2. 기간 필터
   - (a) 이력을 날짜 범위로 필터 (매치 일시 기준인지 수정 일시 기준인지 와이어프레임에 명시 없음)
   - (b) 날짜 2개 입력 후 돋보기 클릭
   - (c) mock: 범위 filter — 기준 필드는 스펙에서 결정(수정 일시 권장)
3. 뒤로가기
   - (a) 매치 결과 관리(17:530)로 복귀
   - (b) "◀ 뒤로가기" 클릭
   - (c) mock: history.back
4. 사이드바 / 로그아웃 — 공통

## 특이사항
- 다른 5개 관리 프레임과 달리 탭 바가 없는 독립 서브페이지 — "뒤로가기"의 목적지가 매치 결과 관리 탭이어야 자연스러움.
- 예시 행이 17:585 수정 모달의 매치(박준서 vs 이종혁, 07-03 19:00)와 연결됨: Player 1 Win → DRAW로 수정한 기록. 다만 수정 일시가 2026-07-09로 매치 일시보다 6일 뒤 — placeholder 값.
- 삭제 이력은 테이블 스키마상 표현 불가(이전/이후 결과 열뿐) — 삭제 로그 포함 여부는 스펙 결정 필요.
