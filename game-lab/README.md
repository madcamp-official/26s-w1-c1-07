# MADPUMP — 로컬 2인 미니게임 (v1 오프라인)

1:1 대결 웹 미니게임. 한 키보드로 두 명이 플레이한다.
P1 = `Q` `W` / P2 = `U` `I` (판정은 `e.code` 기준이라 한글 IME 상태 무관)

## 실행

```bash
npm install
npm run dev      # http://localhost:5173
npm run build
```

## 구조 (기술 스택 결정서 준수)

```
shared/   게임 코어 = 순수 로직 (state, inputs, dt) => newState — DOM/소켓 의존 없음
client/   Vite + React. React는 화면 전환/오버레이만, 게임 루프는 Canvas 2D + rAF
```

온라인 모드 추가 시 `server/` 워크스페이스를 만들어 같은 `shared/` 코어로 서버 권위 판정.

## 게임 규칙 정본 (이 레포 기준)

공통: 진입 즉시 "START !" 표시와 함께 시작(레디 카운트 없음) → 10초 카운트다운 →
결과(P1 승/P2 승/DRAW) 3초 표시 → 메인 복귀. 결과 표시 중 입력 무시.

**게임1 숫자 맞추기** — 타겟/시작 숫자 [1,100] 랜덤(타겟≠시작 숫자, 거리 제한 없음).
Q/U −1, W/I +1, [1,100] 클램프, 키 반복(홀드) 무시.
**타겟에 정확히 도달하는 순간 즉시 승리.** 10초 만료 시 더 가까운 쪽 승, 같으면 DRAW.

**게임2 로켓 피하기** — P1 발사대가 상단을 SCAN 왕복(Q 방향 반전, W 발사·쿨타임 0.25초,
동시 로켓 수 무제한). 로켓 낙하 속도·P2 이동 속도는 **게임 시작 시 1회 랜덤, 게임 내내 고정**.
피격 즉시 P1 승, 10초 생존 시 P2 승.

**게임3 펜싱** — 줄다리기 모델: 넉백당한 만큼 상대가 전진(사거리 개념 없음).
공격 지속 0.1초 / 회피 지속 0.2초 / 각 버튼 쿨타임 0.5초.
공격 구간에 상대 회피가 겹치면 공격자 넉백(PARRIED), 안 겹치면 피격자 넉백(HIT),
회피 구간에 상대 공격이 없었으면 회피 종료 시점에 소급해 회피자 넉백(WHIFF).
같은 프레임의 반대 방향 동시 넉백은 상쇄(무효). 링아웃 즉시 승리,
10초 만료 시 접점이 상대 진영에 있는 쪽 승, 정중앙이면 DRAW.

## 기술 스택 결정서(madpump-tech-stack.md)와 다른 점

규칙 충돌은 팀 논의 결과를 우선한다는 결정에 따름 (2026-07-03):

1. **게임1**: 결정서의 "타겟 일치 3초 유지 시 승리" → **일치 즉시 승리**로 변경.
2. **게임3**: 결정서의 "1초 틱 가위바위보(공격/회피/무행동)" → **실시간 쿨타임·지속시간
   기반 판정**으로 변경. ⚠️ 결정서는 게임3을 "넷코드 최하 난이도"로 보고 온라인 1순위로
   잡았는데, 실시간 판정으로 바뀌면서 이 전제가 깨짐 — 온라인화 시 재검토 필요.
3. 라운드 수/시간 설정(방장 기능)은 온라인 기능이므로 v1 오프라인에서는 10초 고정.
